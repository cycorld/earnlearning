# 2주차: 나의 문제에서 사업 기회 찾기

> **날짜**: 2026년 3월 13일 (금)
> **키워드**: 좋은 문제 정의, 문제 정의 프레임워크, AI 드리븐 시장 분석, 솔루션 스케치

---

## 수업 목표

- "좋은 문제" 의 조건 (내 문제 / 빈도 / "왜 아직 없지?") 을 이해한다
- `[누가] + [언제/어디서] + [무엇을 하려는데] + [어떤 불편이 있다]` 프레임워크로 문제를 구체적으로 정의한다
- AI 에게 시장 분석을 시키는 프롬프트 패턴을 익힌다
- 본인의 문제 3가지를 정리해 첫 번째 MVP 후보를 좁힌다

> "모든 위대한 제품은 누군가의 불편함에서 시작되었다."

## Part 1: 강의 (30분)

### 1. "좋은 문제" 란 무엇인가? (10분)

3가지 판별 기준:

| 기준 | 핵심 |
|---|---|
| **내 문제 vs 남의 문제** | 직접 겪는 문제일수록 솔루션의 깊이가 다릅니다. **공감이 곧 경쟁력.** |
| **크기보다 빈도** | 매일 겪는 작은 불편이 가끔 겪는 큰 불편보다 사업성이 높습니다. |
| **"왜 아직 없지?"** | 이 질문이 떠오르는 순간이 사업 기회의 시작. 그 느낌을 놓치지 마세요. |

> **"스타트업의 시작은 거창한 아이디어가 아니라, 해결되지 않은 작은 불편입니다."**

### 2. 문제 정의 프레임워크 (10분)

```
[누가] + [언제/어디서] + [무엇을 하려는데] + [어떤 불편이 있다]
```

**예시**:

> "대학생이 + 팀플할 때 + 일정 조율하려는데 + 카톡으로 하면 이전 내용이 묻힌다"

→ **솔루션**: 팀플 전용 일정 조율 & 히스토리 관리 서비스

> 불편의 핵심을 뒤집으면, 솔루션이 됩니다.

#### 좋은 문제 vs 아쉬운 문제

| 좋은 문제 | 아쉬운 문제 |
|---|---|
| "매일 아침 옷 고르는 데 20분 걸려" → 구체적, 빈도 높음, 측정 가능 | "세상을 바꾸고 싶어" → 너무 추상적, 측정 불가 |
| "동아리 회비 관리가 엑셀로는 한계" → 특정 타겟, 명확한 불편, 대안 존재 | "AI 플랫폼을 만들고 싶어" → 솔루션부터 시작, 문제가 없음 |

> 문제가 구체적일수록, 솔루션도 구체적이 됩니다.

### 3. AI 드리븐 시장 분석 라이브 시연 (10분)

AI 에게 문제를 던지면, 시장 분석이 돌아옵니다.

**프롬프트 템플릿**:
```
나는 [문제 상황]을 겪고 있어.
이 문제를 해결하는 기존 서비스가 있는지 찾아주고,
없다면 어떤 형태의 솔루션이 가능한지 제안해줘.
타겟 사용자와 시장 규모도 분석해줘.
```

**AI 가 분석해주는 것**:
- 경쟁 서비스
- 시장 규모
- 차별화 포인트
- 타겟 사용자

라이브 시연: 교수가 본인 문제 1개로 ChatGPT/Claude/Gemini 에 동시 질의 → 응답 비교.

## Part 2: 실습 (80분)

### 실습 ① 나의 불편 리스트업 (15분)

1주차 브레인스토밍 20개를 꺼내서, 영역별로 분류하세요. 가장 자주 겪는 불편 **TOP 5** 를 선정합니다.

| 일상 | 학교 | 동아리 | 아르바이트 | 취미 |

→ 각 영역에서 **최소 3개씩** 불편한 점을 적고, 전체에서 TOP 5 선정.

### 실습 ② AI 와 함께 시장 분석 (30분)

TOP 5 중 **3개**를 골라 AI 에게 시장 분석을 요청.

| 항목 | 내용 |
|---|---|
| 경쟁 서비스 | 이미 존재하는 유사 서비스 파악 |
| 기존 서비스 한계 | 기존 서비스가 해결 못하는 부분 |
| 나만의 차별점 | 내가 만들면 다를 수 있는 점 |
| 타겟 사용자 | 누가 가장 필요로 할까? |

→ 문제 3개 × AI 분석 = **나의 시장 분석 보고서** 완성.

### 실습 ③ 솔루션 스케치 (15분)

3개 문제 중 **최종 1개**를 선정하세요. 이것이 여러분의 첫 번째 MVP 가 됩니다.

| 1. 한 문장 솔루션 | 2. 핵심 기능 3가지 | 3. 타겟 사용자 |
|---|---|---|
| "OOO 를 위한 OOO 서비스" | 이 서비스가 꼭 해야 하는 것 | 누가 가장 먼저 쓸 것인가 |

→ 선택한 문제를 한 문장으로 정의하고, 핵심 기능 3가지를 나열.

### 발표 + 피드백 (20분)

- 자원자 3-5명이 본인의 문제 + 솔루션 1분 발표
- 동료/교수 피드백 받기
