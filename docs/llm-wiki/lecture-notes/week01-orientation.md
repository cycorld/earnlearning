# 1주차: 오리엔테이션 — 바이브 코딩 시대의 시작

> **날짜**: 2026년 3월 6일 (금)
> **키워드**: 바이브 코딩, 1인 창업, 한 학기 결과물 4개 마일스톤, AI 도구 (Gemini/Claude/ChatGPT), 라이브 LMS 시연

---

## 수업 목표

- 왜 지금 코딩을 배우는가 — 바이브 코딩 시대의 의미를 이해한다
- 한 학기 동안 무엇을 만들고 무엇을 얻을지 4가지 마일스톤을 인지한다
- 사용할 AI 도구를 직접 계정 생성하고, 다음 시간까지 준비한다
- 글쓰기 3종 (자기소개 / 브레인스토밍 / 미래 예측) 을 통해 본인의 출발점을 명문화한다

---

## Part 1: 왜 이 수업인가 (20분)

### 1. 시대 진단 (5분)

> "미래를 예측하는 가장 좋은 방법은 직접 만드는 것이다."

2026년, 우리는 **바이브 코딩(Vibe Coding) 시대**에 진입했습니다.

- 자연어로 의도를 설명 → AI 가 코드를 작성
- 코딩 경험이 전혀 없어도, **6주 만에 IT 서비스를 만들고 출시** 가능
- 이 수업의 핵심: **나의 문제를 해결하는 솔루션 만들기**

### 2. 새로운 패러다임 (5분)

| 과거 | → | 지금 |
|---|---|---|
| 몇 달간 문법 공부 | | 말로 원하는 것을 설명 |
| 끝없는 문서 읽기 | | AI 가 실제 서비스 코드 작성 |
| 난해한 에러 메시지와 씨름 | | AI 가 에러를 찾아 수정 |
| 토이 프로젝트만 가능 | | 몇 주 만에 실제 서비스 출시 |

**숫자로 보는 변화**:
- **1**: 혼자서도 완성하는 제품 개발
- **6**: 아이디어에서 출시까지 단 6주
- **0**: 사전 코딩 경험 필요 없음
- **2+**: 이번 학기에 배포할 MVP 수

### 3. 1인 창업 시대 (10분)

한 사람이 무한한 가능성을 가진 시대:

| 단계 | 내용 |
|---|---|
| 아이디어에서 MVP로 | 문제 정의 + 솔루션 설명 → AI 가 실제로 제작 |
| 자동화 마케팅 | SNS 콘텐츠 자동 생성, 유튜브 쇼츠, 마케팅 파이프라인 구축 |
| PMF 탐색 | 출시 → 측정 → 학습 → 피봇. 실제 사용자 데이터로 빠르게 반복 |

> **"팀이 필요한 게 아닙니다. 비전과 올바른 도구가 필요합니다."**

타겟 사용자가 적어도 괜찮습니다. 가족, 동아리, 커뮤니티. 중요한 건 **실제로 작동하는 것을 만드는 것**입니다.

## Part 2: 한 학기 여정 (15분)

### 4. 한 학기 결과물 4가지 (5분)

| # | 결과물 | 설명 |
|---|---|---|
| 01 | **2개 이상의 실제 배포된 MVP 서비스** | 사람들이 실제로 사용할 수 있는 인터넷에 공개된 제품 |
| 02 | **정부 지원사업 신청 가능한 사업계획서** | 창업 지원금 신청에 활용할 수 있는 수준 |
| 03 | **투자용 IR 덱 & 3분 피치 스크립트** | 투자자 대상 발표 자료 + 엘리베이터 피치 |
| 04 | **자동화된 마케팅 파이프라인** | 콘텐츠 생성부터 배포까지 자동 운영되는 시스템 |

### 5. 15주 강의 구성 (10분)

| 주차 | 단계 | 내용 | 산출물 |
|---|---|---|---|
| 1-8주 | **기획 & 개발** | 사업 기획 (RFP, PRD, SPEC), PR&FAQ 작성법, 첫 MVP 개발, Cursor + Claude 바이브 코딩, Vercel/Netlify 배포 | 1차 MVP 배포 |
| 9-12주 | **마케팅 & 반복** | 자동화 마케팅 시스템, SNS 콘텐츠 자동 생성, Google Analytics & PMF 검증, 사용자 피드백 수집, 두 번째 MVP 개발 | 2차 MVP + 마케팅 |
| 13-15주 | **피칭 & 런칭** | 정부과제용 사업계획서 작성, IR 덱 제작, 3분 엘리베이터 피치, 데모데이 발표, 네트워킹 | 사업계획서 + IR 덱 |

## Part 3: AI 도구 소개 (15분)

### 6. 앞으로 사용할 주요 도구

| 도구 | 강점 |
|---|---|
| **Gemini** | 구글 모델. 유튜브 연계 잘됨. 강력한 Nanobanana, Veo. 프론트엔드 디자인 강점. 가장 큰 컨텍스트 윈도우 |

## Part 4: 평가 + 운영 (20분)

### 7. 평가 방식 (5분)

학습평가방식 (절대평가 속 상대평가):

| 비중 | 항목 |
|---|---|
| 50% | 프로젝트 |
| 20% | 과제물 |
| 20% | 발표 |
| 10% | 참여도 |

**4가지 핵심 마일스톤**:
1. 1차 MVP 배포 (8주차까지)
2. 2차 MVP 배포 (12주차까지)
3. 사업계획서 제출 (13주차)
4. IR 덱 & 최종 발표 (15주차)

→ 4가지 모두 완료 = **A그룹**, 3가지 = **B그룹**, 2가지 = **C그룹**, 1가지 = **D그룹**.
→ 그룹 내 상대평가 (+, 0, -)

### 8. 수업 운영 방식 (10분)

본 수업은 **AI 활용을 적극 권장**합니다. 모든 과제와 프로젝트에서 AI 도구 사용이 허용됩니다.

| 시간 비중 | 내용 |
|---|---|
| 강의 (20%) | 30-60분 핵심 개념, 프레임워크/전략 소개, 라이브 시연 |
| 실습 (60%) | 바이브 코딩 실습, 개별 프로젝트 코칭, AI 와 페어 프로그래밍 |
| 토론 (20%) | 학생 발표, 상호 피드백 세션, 데모데이 리허설 |

> **핵심 역량은 코드를 작성하는 것이 아니라, AI 가 생성한 결과를 검토하고 개선하는 능력입니다.**

### 9. 기술투자 협업 모델 (5분)

Free tier 는 한계가 많습니다. 세 가지 길:

| 모델 | 설명 |
|---|---|
| **스스로 투자** | 최소 하나 이상 유료 모델 사용. 지출 이상을 벌 수 있다는 마인드. 교육에 대한 투자 |
| **외부 지원** | 자신의 비전을 설명하고 후원자를 찾기. Take & Give |
| **엔트로피 패러독스 투자** | 토큰 소모가 큰 작업을 교수 회사 시스템(EarnLearning)으로 지원. Give & Take |

> 지금, 과도기에는 다들 비슷한 출발선이지만, 몇 년이 지나면 부익부 빈익빈이 고착화 됩니다. 언제나 그랬듯이.

## Part 5: 라이브 시연 — LMS 만들기 (30분)

### 10. 시연 목표 + 첫 프롬프트 (5분)

지금부터 여러분이 사용할 **수업 관리 시스템(LMS)** 을 라이브로 개발합니다.

- **오늘의 목표**: 수업이 끝나기 전에 LMS 를 완성하고, 모든 학생이 회원가입하고 가기
- **왜 직접 만드나**: 바이브 코딩이 실제로 어떻게 작동하는지 눈앞에서 보여드리기 위해. AI 에게 말로 설명하면 코드가 나옵니다.

**기술 스택**:
- Backend: Go + SQLite3
- Frontend: HTML + HTMX
- Deploy: 실시간 서버 배포

### 11. AI 에게 보낼 첫 프롬프트

```
대학 수업용 간단한 LMS를 만들어줘.
백엔드는 Go + SQLite3 + minio, 프론트는 HTML + HTMX.
학생이 회원가입/로그인하고, 매주 글(사진 첨부)을 작성해서
제출할 수 있는 시스템이야.
교수는 학생 글을 확인하고 피드백을 남길 수 있어야 해.
```

### 12. 기능 명세 리스트

| 학생 기능 | 교수 기능 | 시스템 |
|---|---|---|
| 회원가입 / 로그인 | 학생 목록 조회 | 주차별 과제 관리 |
| 주차별 글 작성 및 제출 | 주차별 제출물 확인 | 마감일 설정 |
| 내 제출 내역 확인 | 피드백 작성 | 반응형 웹 디자인 |
| 교수 피드백 확인 | 제출 현황 대시보드 | 실시간 서버 배포 |

> 여러분이 글쓰기 실습하는 동안, 교수가 실시간으로 LMS 를 개발합니다!

## Part 6: 오늘의 실습 — 글쓰기 3종 (40분)

LMS 가 완성되는 동안, 학생들은 아래 3가지 글쓰기를 진행합니다.

### 실습 ① 자기소개 (10분)

자기 자신에 대해 자유롭게 써보세요. 형식은 상관없습니다 — 편지, 에세이, 리스트.

**이런 내용을 담아보세요**:
- 내가 요즘 관심 있는 것
- 내가 잘하는 것
- 내가 고민하는 것
- 이 수업에서 기대하는 것
- 졸업 후 하고 싶은 것

### 실습 ② 나혼자 브레인스토밍 (15분)

> "내가 개발이 가능하다면, 이런 것들을 만들고 싶다."

비판 금지! 아무 제약 없이 자유롭게 나열하세요.

**규칙**:
- 현실성 판단 금지
- 기술적 가능성 판단 금지
- "이건 안 되겠지" 금지

→ **최소 20개 이상 작성**
