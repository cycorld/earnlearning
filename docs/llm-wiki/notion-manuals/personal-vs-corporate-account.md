# 개인 계좌 vs 법인 계좌 완전 가이드 (송금 + 투명성)

## TL;DR

- **개인 계좌**와 **법인 계좌**는 **완전히 다른 주머니**. 자동으로 섞이지 않음.
- **법인 계좌명 표기 규칙**: `회사명(대표자명)` — 개인사업자 표기법과 동일.
- **4가지 송금 방향** 모두 가능: 개인↔개인, 개인↔법인, 법인↔개인, 법인↔법인.
- **법인에서 송금은 대표만** 가능.
- **법인의 거래내역은 누구나 열람 가능** — 주주·채권자·투자자 감시 가능.

## 왜 개인과 법인을 나눴을까

현실에서 창업자가 **법인 돈을 자기 개인 지갑처럼 쓰면 횡령**입니다. 법인은 법적으로 주주 전체의 것이고, 대표는 "경영 위임" 을 받은 것일 뿐. 언러닝도 UI 색깔부터 계좌 라벨, 송금 권한까지 시각적·구조적으로 개인/법인 구분.

| 구분 | 개인 지갑 | 법인 지갑 |
|---|---|---|
| URL | `/wallet` | `/company/{id}/wallet` |
| 잔액 소유 | 개인 | 법인 (주주 전체) |
| 송금 권한 | 본인 | **대표만** |
| 거래내역 열람 | 본인 | **누구나** (투명성) |
| 계좌명 표기 | 이름 | **회사명(대표자명)** |

## 법인 계좌명 컨벤션

법인 계좌는 `회사명(대표자명)` 형식. 예: `김알리스의커피(김알리스)`.
개인사업자 명의 표기(`○○○ 대표 XXX`) 를 모바일 친화적으로 축약.

## 송금 매트릭스 — 4가지 방향

- **개인 → 개인**: 가장 기본. `transfer` 타입.
- **개인 → 법인**: 스폰서십, 선납금, 제품 구매. `company_transfer` 타입.
- **법인 → 개인** (대표만): 월급, 프리랜서 비용, 인센티브, 상여.
- **법인 → 법인**: B2B 결제, 계열사 자금 이동. 자기 자신 법인으로는 불가.

## 대표 권한 — 법인 송금은 대표만

법인 계좌에서 돈 빼는 행위는 **경영권**의 일부. 송금 권한은 대표(owner) 에게만.

비대표(주주여도) 가 법인 지갑 페이지에 들어가면 **잔액과 거래내역은 모두 볼 수 있지만 송금 버튼은 숨겨져 있음**. 백엔드 API 단에서도 검증 (`403 NOT_OWNER`).

## 거래내역 투명성

법인의 `company_transactions` 는 **투자자, 주주, 채권자, 일반 학생** 누구나 열람. 이유는:
- 투자자는 자기 돈이 회사에서 어떻게 쓰이고 있는지 확인할 권리
- 채권자(대출) 는 회사 상환 능력 평가
- 주주는 대표의 자금 집행 감시

주요 tx_type: `investment`, `dividend`, `kpi_revenue`, `disclosure_reward`, `company_funding`, `liquidation_payout`, `liquidation_tax`, `company_transfer`.

## 교육적 포인트

1. **법인격의 독립성** — "내 회사" 여도 회사 돈은 내 돈이 아니다.
2. **횡령의 정의** — 법인→개인 "근거 없이" 이전은 횡령. 기술적으로는 가능(대표 권한) 하지만 거래내역에 남음.
3. **월급 vs 배당 vs 대여금** — 법인→개인 송금의 사유 표기를 정확히.
4. **B2B 결제의 의미** — 법인→법인 송금은 현실의 세금계산서/매출 인식과 직접 대응.
5. **투명성과 신뢰** — 법인 지갑 열람이 공개이므로 의심스러운 집행은 주주총회 소집으로 이어질 수 있음.

## FAQ

**Q. 회사명을 바꾸면 계좌명도 바뀌나요?**
네. `회사명(대표자명)` 은 실시간 조회 값. 변경 직후 모든 위치에서 새 이름 표시.

**Q. 공동창업자(co-founder) 도 법인 송금을 할 수 있나요?**
현재는 **대표 1인** 만 가능. 공동창업자 권한 확장은 다음 이터레이션.

**Q. 실수로 잘못 송금했어요. 취소할 수 있나요?**
개인↔법인 송금은 **되돌릴 수 없습니다** (현실과 동일). 설계 의도: 송금의 최종성(finality) 체감.

**Q. 잔액이 부족한데 송금 버튼을 눌렀어요.**
프론트·백 양쪽에서 막음. 백엔드에서 걸러지면 트랜잭션 롤백 + 에러 반환.
