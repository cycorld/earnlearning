# 대출 0% 이자율 허용: 무이자 대출이 안 되던 버그 수정

**날짜**: 2026-03-17
**태그**: `버그수정`, `대출`, `은행`

## 무엇을 했나요?

대출 승인 시 0% 이자율을 입력해도 강제로 5%가 적용되던 버그를 수정했습니다.

## 왜 필요했나요?

관리자가 무이자 대출(0%)을 승인하려고 했는데, 백엔드에서 `interestRate <= 0`이면 기본값 5%로 덮어쓰는 로직이 있었습니다. 0은 유효한 값이므로 음수만 차단하도록 수정했습니다.

## 어떻게 고쳤나요?

```go
// 변경 전: 0도 기본값으로 대체됨
if interestRate <= 0 { interestRate = 5 }

// 변경 후: 음수만 차단, 0은 허용
if interestRate < 0 { interestRate = 0 }
```

추가로 프론트엔드 승인 다이얼로그에 기본값 5%를 채워넣고, "주간 이자율"이라는 잘못된 라벨을 "연 이자율"로 수정했습니다.

## 사용한 프롬프트

```
은행이자가 5% 고정인거 같아. 승인 할 때 프론트에서 입력한 숫자가 반영안된거 같아. 관련 부분들 리뷰해줘.
아, 내가 0% 넣어서 그랬네. 0도 가능하게 해줘. 그리고 기본값을 채워 넣어줘.
```

## 배운 점

경계값 처리에서 `<= 0`과 `< 0`의 차이는 크다. 0이 유효한 입력인 경우 `<= 0`으로 검사하면 의도치 않게 기본값으로 대체된다.
