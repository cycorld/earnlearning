# 136. 투자처 목록이 20개까지만 보이던 문제 + 카드가 붙어 보이던 문제

**날짜**: 2026-06-29
**티켓**: #136
**타입**: 버그수정 / UX

## 무엇을 했나요?

투자 탭의 **투자 라운드(투자처) 목록**에서 두 가지를 고쳤습니다.

- **A. 21번째부터 안 보이던 문제**: 오픈된 투자처가 20개를 넘으면 21번째부터는 화면에 나오지 않았습니다. 이제 **전부** 보입니다.
- **B. 카드가 세로로 딱 붙어 보이던 문제**: 라운드 카드들 사이에 간격이 없어 답답했는데, 카드 사이 여백을 되살렸습니다.

## 왜 필요했나요?

### A — 페이지네이션
백엔드의 목록 API(`GET /investment/rounds`)는 한 번에 너무 많은 데이터를 주지 않으려고 **페이지 단위로 끊어서** 응답합니다. 파라미터를 안 주면 기본값이 "한 페이지 20개"라서, 프론트가 그냥 `?status=open` 만 요청하면 **첫 20개만** 받아왔습니다. 응답에 같이 오는 전체 개수(`total`)도 쓰지 않아, 나머지를 더 가져올 생각을 못 했습니다.

### B — inline 요소에는 세로 여백이 안 먹는다
카드 사이 간격은 Tailwind 의 `space-y-3`(= 형제 요소들에 위쪽 바깥여백을 줌)으로 줬습니다. 그런데 라운드 카드만 **링크(`<Link>` → `<a>`)로 감싸** 있었고, `<a>` 는 기본이 **inline** 요소입니다. inline 요소에는 위/아래 바깥여백(margin)이 적용되지 않아 간격이 사라졌습니다. (포트폴리오·배당금 탭은 카드를 링크로 감싸지 않아 멀쩡했습니다.)

## 어떻게 만들었나요?

### A
- `fetchAllOpenRounds()` 를 만들어 **페이지를 순회**합니다. 한 페이지를 `limit=50`(백엔드 허용 최대치 — 그보다 크게 주면 20으로 깎임)으로 요청하고, 모은 개수가 응답의 `total` 에 도달할 때까지 다음 페이지를 이어서 받습니다. 무한 루프를 막는 50페이지 안전 상한도 뒀습니다.

### B
- 라운드 카드를 감싼 `<Link>` 에 `className="block"` 한 줄 추가. inline 이던 앵커가 block 이 되면서 `space-y-3` 의 세로 간격이 정상 적용됩니다.

## 배운 점

- **목록 API 를 쓸 땐 항상 "전부 다 받았나?"를 의심하자.** 페이지네이션이 있는 API 는 기본값이 작게 잡혀 있어서, 응답의 `total` 과 받은 개수를 비교해 더 가져와야 할 때가 많습니다.
- **CSS 간격 유틸은 요소의 display 를 탄다.** `space-y-*` 는 margin 기반이라 `inline` 요소(특히 링크 `<a>`)엔 안 먹습니다. 클릭 영역을 위해 카드를 링크로 감쌀 땐 `block`(또는 `flex`)으로 만들어 줘야 레이아웃이 의도대로 동작합니다.
