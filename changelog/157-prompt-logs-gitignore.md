# 실수로 DB가 공개 저장소에 올라가지 않도록 안전장치 추가

## 무엇을 했나요?

두 가지 정리 작업을 했습니다.

1. `.gitignore`에 `backend/data/`(로컬 개발용 SQLite DB와 업로드 파일)와 `.claude/worktrees/`(AI 에이전트 작업 폴더)를 추가했습니다.
2. 그동안 쌓여 있던 프롬프트 캡처 로그 13건(`docs/prompts/011~021`, `backend/docs/prompts/001~002`)을 보안 점검 후 커밋했습니다.

## 왜 필요한가요?

이 저장소는 **공개(public)** 저장소입니다. `backend/data/` 안에는 실제 학생 데이터가 담긴 SQLite DB 파일이 있는데, 이 폴더가 git에서 무시(ignore)되지 않은 채 "untracked" 상태로 남아 있었습니다. 개별 `*.db` 파일은 이미 무시 규칙이 있었지만, 폴더 안의 `uploads/` 디렉토리는 규칙에 걸리지 않아 `git add -A` 한 번이면 실수로 커밋될 수 있는 상태였습니다. 사고가 나기 전에 폴더 전체를 무시하도록 안전장치를 걸었습니다.

## 어떻게 만들었나요?

- `git status`로 untracked 파일을 전수 조사하고, `git check-ignore`로 무시 규칙이 실제로 적용되는지 확인했습니다.
- 프롬프트 로그 13건은 커밋 전에 한 건씩 열어 학생 실명·학번·이메일·API 토큰 포함 여부를 점검했습니다(보안 규칙 #102). 전부 운영자 지시 프롬프트만 담겨 있어 안전했습니다.

## 사용한 프롬프트

> 응 권장대로 gitignore 추가 해주고 프롬프트는 보안 점검 후 커밋하고 배포해줘.

## 배운 점

`*.db` 같은 파일 단위 무시 규칙만으로는 부족할 수 있습니다. 데이터가 담기는 **폴더 자체**를 무시해야 폴더 안에 새로 생기는 어떤 파일이든 안전합니다. 공개 저장소에서는 "지금 안 올라갔으니 괜찮다"가 아니라 "실수해도 안 올라가는 구조"를 만들어야 합니다.
