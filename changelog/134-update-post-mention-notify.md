# 134. 게시글 수정 시 신규 멘션 알림

**날짜**: 2026-06-16
**티켓**: #134
**타입**: 버그 수정

## 무엇을 했나요?

게시글을 **수정**하면서 새로 추가한 `@멘션`도 알림이 가도록 고쳤습니다. 이때 **새로 추가된 멘션에만** 알림을 보내고, 원래 본문에 이미 있던 멘션에는 다시 보내지 않습니다.

## 왜 필요했나요?

#132에서 멘션 알림은 글/댓글을 **새로 쓸 때**(`CreatePost`, 댓글 생성)만 발송됐습니다. 글을 나중에 고치며 누군가를 멘션으로 추가하면, 그 사람은 알림을 못 받았습니다. 실제로 한 게시글을 `@최용철` → `@[최용철](user:1)` 정식 멘션 형식으로 수정했는데 대상에게 알림이 안 가서 발견했습니다.

단순히 "수정 때도 멘션 알림 보내기"로 끝내면 함정이 있습니다. 글을 한 글자 고칠 때마다 본문의 **모든** 멘션 대상에게 알림이 다시 가서 **스팸**이 됩니다.

## 어떻게 만들었나요?

"수정 전 본문에 있던 멘션"을 제외 목록(exclude)으로 만들어, 새 본문의 멘션 중 **거기에 없던 사람에게만** 알림을 보냅니다 (집합 차집합).

- `notifyMentions(...)`에 `exclude map[int]bool` 인자를 추가. 생성 경로(글/댓글)는 `nil` 전달로 기존 동작 유지.
- `UpdatePost`는 수정 직전 본문의 `extractMentions()` 결과를 exclude로 넘김.

### TDD

먼저 실패 테스트(`TestUpdatePostMentionNotification`)를 작성했습니다:
1. 수정으로 추가된 멘션 대상 → 알림 1건
2. 원래 있던 멘션 대상 → 재알림 없음(여전히 1건)
3. 같은 본문으로 다시 저장 → 신규 알림 없음

Red(추가된 멘션 알림 0건) 확인 후 구현해 Green 전환, 기존 멘션 테스트 전체 통과 확인.

## 배운 점

- **"기능을 켜는 것"과 "스팸을 안 만드는 것"은 별개**: 수정 알림은 "보낼까 말까"가 아니라 "누구에게 보낼까"가 핵심이었습니다. 이전 상태와의 **차이(diff)** 가 알림의 단위입니다.
- 함수에 인자 하나(`exclude`)를 더해 생성/수정 두 경로가 같은 알림 로직을 공유하게 하면, 멘션 검증·미리보기·자기멘션 제외 규칙이 한 곳에만 있어 어긋날 일이 없습니다.
