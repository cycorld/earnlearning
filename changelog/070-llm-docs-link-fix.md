# 070. /llm 페이지 "API 문서 보기" 링크 수정

**날짜**: 2026-04-18
**태그**: 프론트엔드, 버그수정, LLM

## 무엇을 했나
`/llm` 페이지 우측 하단의 **"API 문서 보기 ↗"** 링크가 `https://llm.cycorld.com/admin/docs`
(관리자 전용 Swagger UI) 로 연결되어 있었는데, 학생에게는 admin key 가 없어서
접속하면 401 이 떴다. 링크 대상을 학생용 공개 랜딩 페이지 `https://llm.cycorld.com/`
로 변경.

## 왜 필요했나
#069 (LLM API 키 학생 프로비저닝) 에서 학생용 페이지를 만들 때, 개발자 관점으로
Swagger URL 을 그냥 붙여버린 게 실수. 학생이 로그인 없이도 볼 수 있는 API 사용
가이드(코드 예시·모델 설명·엔드포인트 목록) 는 `/admin/docs` 가 아니라 루트 랜딩에
이미 다 있다.

## 어떻게 만들었나
`frontend/src/routes/llm/LlmPage.tsx` 에서 href 한 줄만 수정. 이 앵커만 바뀐 1줄
변경이라 테스트는 스모크로 충분.

## 배운 점
- **"보이는 링크는 로그인 없이 직접 눌러보기"** 가 배포 전 체크리스트의 기본.
  특히 외부 서비스 링크는 권한 경계가 다를 수 있어서, 개발자 환경에선 보이는데
  학생 계정에선 막히는 패턴이 생기기 쉽다.
- 어드민 키는 서버 env 에만 두는 원칙은 잘 지켜졌는데, 정작 학생에게 어드민 페이지
  링크를 UI 에 노출시킨 건 그 자체로는 보안 이슈는 아니지만 UX 깨지는 포인트.
