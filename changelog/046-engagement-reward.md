# 046. 게시글 좋아요/댓글 보상 시스템

**날짜**: 2026-04-11
**티켓**: #008
**타입**: 신규 기능

## 무엇을 했나요?

게시글에 좋아요나 댓글이 달리면 글쓴이에게 가상화폐 보상이 자동으로 지급되는 시스템을 만들었습니다.

## 왜 필요했나요?

학생들이 서로의 게시글에 적극적으로 반응하도록 인센티브를 제공하기 위해서입니다. 좋은 글을 쓰면 다른 학생들의 좋아요와 댓글을 통해 보상을 받을 수 있고, 이를 통해 커뮤니티 활성화와 학습 동기부여를 동시에 달성합니다.

## 어떻게 만들었나요?

### 보상 규칙
| 행동 | 보상 | 조건 |
|------|------|------|
| 좋아요 | 글쓴이에게 +10원 | 자기 글 제외 |
| 좋아요 취소 | 글쓴이에게서 -10원 회수 | |
| 댓글 작성 | 글쓴이에게 +100원 | 자기 글 제외 |
| 댓글 삭제 | 글쓴이에게서 -100원 회수 | |

### Backend 변경

1. **트랜잭션 타입 추가**: `like_reward`, `comment_reward` 두 가지 새로운 거래 유형을 등록했습니다.

2. **LikePost 수정**: 기존 좋아요 토글 로직에 보상 지급/회수를 추가했습니다. 자기 글에 좋아요하면 보상이 없고, 다른 사람의 글이면 10원이 글쓴이 지갑에 입금됩니다.

3. **CreateComment 수정**: 댓글 작성 시 자기 글이 아니면 글쓴이에게 100원 보상을 지급합니다.

4. **댓글 삭제 API 신규**: `DELETE /posts/:id/comments/:commentId` 엔드포인트를 새로 만들었습니다. 본인이 쓴 댓글만 삭제 가능하고, 삭제 시 글쓴이에게 줬던 보상 100원을 회수합니다.

5. **헬퍼 함수**: `creditOrDebitUser()` - 지갑이 없는 사용자에게 보상할 때 자동으로 지갑을 생성해주는 유틸리티 함수입니다.

### Frontend 변경

- 좋아요 클릭 시 보상 토스트 알림 표시
- 댓글 작성 시 보상 토스트 알림 표시
- 댓글 옆에 삭제 버튼 추가 (본인/관리자만 표시)

### TDD로 개발

8개 시나리오의 통합 테스트를 먼저 작성하고 구현했습니다:
- 다른 사람 글 좋아요 → 10원 보상
- 좋아요 취소 → 10원 회수
- 자기 글 좋아요 → 보상 없음
- 다른 사람 글 댓글 → 100원 보상
- 자기 글 댓글 → 보상 없음
- 댓글 삭제 → 100원 회수
- 타인 댓글 삭제 시도 → 거부
- 자기 글에 쓴 자기 댓글 삭제 → 차감 없음

## 배운 점

- **TDD의 힘**: 테스트를 먼저 작성하면 "자기 글에 자기가 좋아요"같은 엣지 케이스를 구현 전에 정의할 수 있어서, 빠뜨리기 쉬운 조건을 놓치지 않습니다.
- **트랜잭션 타입 설계**: 보상과 회수를 같은 타입(`like_reward`)으로 기록하면, 나중에 거래 내역에서 필터링하기 편합니다. 금액의 부호(+/-)로 지급/회수를 구분합니다.
