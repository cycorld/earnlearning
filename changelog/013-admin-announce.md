# 공지 알림 기능: 관리자가 전체 유저에게 알림 보내기

**날짜**: 2026-03-15
**태그**: `알림`, `푸시`, `관리자`, `WebPush`, `API`

## 무엇을 했나요?

관리자가 전체 유저(또는 특정 유저)에게 **공지 알림**을 보낼 수 있는 기능을 만들었습니다. 알림은 앱 내 알림 + 웹 푸시 알림으로 동시에 전송됩니다.

## 왜 필요했나요?

기존에는 관리자가 학생들에게 공지를 보낼 방법이 없었습니다. 수업 안내, 과제 마감 알림, 시스템 점검 공지 등을 실시간으로 전달할 수 있어야 했습니다.

## 어떻게 만들었나요?

### 1. 백엔드 API 추가

`POST /api/admin/notifications/announce` 엔드포인트를 만들었습니다.

```json
{
  "title": "공지 제목",
  "body": "공지 내용",
  "user_ids": []  // 비어있으면 전체 유저
}
```

요청을 받으면:
1. `user_ids`가 비어있으면 → DB에서 승인된 전체 유저 ID를 조회
2. 각 유저에게 알림 생성 (DB 저장)
3. WebSocket으로 실시간 알림 전송
4. Web Push로 푸시 알림 전송

### 2. 프론트엔드 공지 페이지

`/admin/announce` 경로에 공지 알림 페이지를 추가했습니다. 제목과 내용을 입력하고 전송 버튼을 누르면 전체 유저에게 공지가 발송됩니다.

### 3. 기존 알림 시스템 활용

새로운 시스템을 만들지 않고, 기존의 `CreateNotification` → WebSocket + Web Push 파이프라인을 그대로 활용했습니다. 이렇게 하면 코드 중복 없이 기존에 검증된 알림 경로를 사용할 수 있습니다.

## 사용한 프롬프트

> admin 기능에 공지 알림 보낼 수 있도록 만들어줘

## 배운 점

- **기존 코드 재활용**: 새 기능을 만들 때 처음부터 다 만들 필요 없이, 기존에 잘 동작하는 코드를 조합해서 만들 수 있습니다.
- **웹 푸시의 동작 원리**: DB 알림 저장 → WebSocket 실시간 전달 → VAPID 키를 이용한 Web Push 전송, 이 세 가지가 함께 동작해야 완전한 알림 시스템이 됩니다.
- **관심사 분리**: Repository(DB) → UseCase(비즈니스 로직) → Handler(HTTP) → Router(경로) 순서로 레이어를 나눠서 각자 역할에 집중하게 했습니다.
