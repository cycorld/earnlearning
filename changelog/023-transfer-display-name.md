# 023. P2P 송금 기능 + 사용자 이름 표시 개선

**날짜**: 2026-03-19
**태그**: `송금`, `지갑`, `UX`, `유틸리티`

## 무엇을 했나요?

1. **P2P 송금**: 자산관리 탭에서 다른 사용자에게 직접 송금할 수 있는 기능
2. **displayName 유틸리티**: 동명이인 구분을 위해 "이름(소속/학번2자리)" 형태로 표시

## 왜 필요했나요?

- 학생 간 자금 이동 수단이 없어서 거래나 협업 시 불편했습니다
- 같은 이름의 학생이 있을 때 누가 누구인지 구분이 어려웠습니다

## 어떻게 만들었나요?

### P2P 송금
- **Backend**: `POST /wallet/transfer` + `GET /wallet/recipients?q=검색어`
- **Frontend**: 지갑 페이지에 "송금하기" 버튼 → 수신자 검색 → 금액 입력 → 송금
- 잔액 부족 시 버튼 비활성화, 자기 자신에게 송금 불가

### displayName 유틸리티
- `displayName({ name, department, student_id })` → "홍길동(컴공/33)"
- 긴 학과명 자동 축약 (컴퓨터공학 → 컴공, 인공지능 → AI)
- 7개 페이지에 적용 (피드, 과제, 외주마켓, 투자, 관리자 등)
- 백엔드: Post/Comment/Grant/Freelance author 객체에 department 필드 추가

## 사용한 프롬프트

```
자산관리 탭에서 송금하기 기능을 넣을 거야. 사용자 중에 선택하거나 회사를 선택해서 송금을 할 수가 있어.
사용자 이름이 겹치는 경우가 있으니 서비스 전체적으로 사용자 이름 표시할 때, "이름(소속/학번2자리)" 식으로 보여주는 유틸짜서 전체적으로 적용해줘.
```

## 배운 점

- **유틸리티 함수 패턴**: `formatMoney`처럼 `displayName`도 공통 유틸로 만들면, 표시 로직 변경 시 한 곳만 수정하면 됩니다.
- **검색 디바운스**: 키 입력마다 API 호출하면 서버 부하가 심해집니다. `setTimeout` + `clearTimeout`으로 300ms 디바운스를 적용했습니다.
- **송금은 두 번의 DB 작업**: 보내는 사람 차감(Debit) + 받는 사람 입금(Credit). 실무에서는 트랜잭션으로 감싸야 하지만, 교육용으로는 순차 실행으로 충분합니다.
