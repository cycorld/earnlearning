# 030. 모바일 PWA 파일 다운로드 수정

> **날짜**: 2026-04-01
> **태그**: `PWA`, `모바일`, `버그수정`, `파일`

## 무엇을 했나요?

모바일 PWA에서 업로드된 파일을 클릭해도 다운로드가 안 되는 문제를 수정했습니다.

## 왜 필요했나요?

PC 브라우저에서는 파일 다운로드가 정상 동작했지만, 모바일(특히 홈 화면에 설치한 PWA)에서는 파일 링크를 클릭해도 아무 반응이 없었습니다.

## 어떻게 만들었나요?

원인 2가지를 수정했습니다:

1. **서비스워커 캐싱 제거**: `/uploads/` 경로에 대한 `CacheFirst` 캐싱 전략이 파일 다운로드 요청을 가로채고 있었습니다. 이를 제거하여 파일 요청이 서버로 직접 전달되도록 했습니다.

2. **PWA standalone 모드 대응**: PWA가 홈 화면에 설치되면 `display: standalone` 모드로 실행됩니다. 이 모드에서는 `target="_blank"` 링크가 시스템 브라우저 대신 앱 내에서 처리되려 하여 파일 다운로드가 트리거되지 않습니다. `window.open()`으로 명시적으로 외부 브라우저를 열도록 수정했습니다.

## 배운 점

- **PWA standalone 모드는 브라우저와 다르다**: `target="_blank"`가 새 탭이 아니라 앱 내에서 처리될 수 있습니다. 파일 다운로드처럼 브라우저 고유 기능이 필요한 경우 `window.open()`을 명시적으로 사용해야 합니다.
- **서비스워커 캐싱은 양날의 검**: 성능을 위한 캐싱이 오히려 파일 다운로드를 방해할 수 있습니다. 동적 콘텐츠(사용자 업로드 파일)는 캐싱 대상에서 제외하는 것이 안전합니다.
