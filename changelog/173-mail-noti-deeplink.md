# 알림을 눌렀는데 홈이 뜬다 — 없는 라우트와 딥링크

## 무엇을 했나요?

메일 수신 알림(푸시/인앱)을 누르면 홈 피드가 떴습니다. 이제 **해당 메일함이 자동 선택되고 그 메일이 바로 열립니다**.

## 왜 그랬나요?

푸시 알림의 클릭 URL이 `/<참조타입>/<id>` 일반 공식으로 만들어져 `/mail/123`이 됐는데, 이런 라우트는 없습니다. SPA의 catch-all(`*` → /feed)이 받아서 홈으로 떨어진 것. "링크가 틀리면 404가 아니라 **엉뚱한 성공**이 된다"는 SPA 특유의 함정입니다.

## 어떻게 고쳤나요?

- 푸시 URL 생성에 알림 타입별 분기 추가: 수신 알림 → `/mail?open=<메일id>`, 주소 승인/반려 → `/mail` (같은 reference_type이라도 id 의미가 다름 — 메일 id vs 주소 id).
- 인앱 알림 목록도 같은 딥링크로.
- 메일함 페이지가 `?open=`을 받으면: 메일 상세를 먼저 조회해 **소속 메일함(address_id)을 알아내 선택**하고 상세를 바로 엽니다. 17개 메일함을 가진 관리자도 클릭 한 번에 정확한 함으로.

## 배운 점

- 알림 클릭 경로는 백엔드(푸시)와 프론트(인앱) **두 곳**에 있습니다. 한쪽만 고치면 반쪽 수정. 프로젝트 알림 체크리스트에 푸시 URL도 포함해야.
- 같은 reference_type 안에서도 notif_type에 따라 id의 의미가 다를 수 있습니다. 딥링크는 일반 공식보다 타입별 명시 매핑이 안전합니다.
