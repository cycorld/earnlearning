# 066. 리스트 카드 간격 추가 정리

**날짜**: 2026-04-18
**태그**: 디자인, 간격, 사용자피드백

## 무엇을 했나
#065 디자인 시스템 리프레시 배포 후 사용자 피드백 — 카드들이 다닥다닥
붙어 보이는 페이지가 있다는 지적. Stage에서 모든 페이지를 실제 브라우저로
순회하며 확인한 결과, 3개 페이지의 카드 리스트 간격이 `space-y-2` (8px) 로
좁게 남아있었음.

## 왜 필요했나
#065 검수 때는 `space-y-3` (12px) 까지만 잡아서 `space-y-4` (16px) 로 올렸고,
`space-y-2` 이하는 미처 잡지 못했음. 사용자가 실제 브라우저에서 순회했을 때
다음 페이지에서 눈에 띄게 답답한 간격이 보였음:

- 알림 페이지 (`NotificationsPage.tsx:217`) — 알림 16개가 답답하게 붙어있음
- 메시지 페이지 (`MessagesPage.tsx:81`) — DM 대화 리스트
- 태스크 보드 (`AdminTasksPage.tsx:100`) — 칸반 카드 리스트

## 어떻게 만들었나
Python 스크립트로 `.map` 호출을 감싸는 부모 `<div>`의 className 수집하고
`space-y-[012]` 로 좁은 간격을 가진 케이스를 모두 나열. 3곳 발견 후 일괄
`space-y-4` 로 승격.

최종 기준: **리스트 형태로 나열되는 카드 컨테이너는 `space-y-4` (16px)** 로
통일. CardContent 내부의 `space-y-3` 은 "카드 내부 요소 그룹"으로 별도 유지.

## 사용한 프롬프트
```
모든 페이지 전수검사 필요한게, 카드들 간격이 없는 페이지가 몇 개 있어.
```

## 배운 점
### 정적 분석만으로는 부족 — 실제 브라우저 검수가 필수
#065 작업 때 `space-y-3` → `4` 는 grep 으로 잡았지만 `space-y-2` 는 놓쳤음.
"카드 리스트의 부모 div className" 이라는 패턴으로 더 넓게 검색했으면 잡혔을
텐데, grep 으로는 의도 파악이 어려움.

정적 스캔은 **규칙 기반으로 놓치는 구석이 있음** — 마지막 단계는 꼭 브라우저
실제 화면 + 사람 눈 (또는 스크린샷 + LLM 판단). 이번엔 사용자가 그 역할을
해줘서 발견. 다음부턴 배포 전 주요 페이지 스크린샷 전수 순회를 루틴화.

### "간격 없음"의 실제 체감은 컴포넌트 밀도에 따라 다름
같은 `space-y-2` 라도 카드 자체가 높이가 크고 내용이 빽빽하면 더 답답함.
알림처럼 카드당 2~3줄 텍스트만 있을 때 특히 심함. 그래서 얇은 카드 리스트는
`space-y-4` 가 안전한 하한.
