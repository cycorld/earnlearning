# 077. 챗봇 답변 품질 개선 — 환각 억제 + 긴 체인 타임아웃 완화

**날짜**: 2026-04-19
**태그**: 챗봇, 프롬프트, 품질

## 이슈 (관찰)
브라우저 E2E 테스트 결과:

1. **환각** — "주주총회 가결 기준?" 질문에 search_wiki 가 LMS 정책(일반 50%, 청산 70%)
   을 반환했는데도 모델이 일반 상법 "4분의 1 출석" 규정을 섞어 답변.
2. **긴 체인 타임아웃** — dev_helper 가 context7_search → context7_docs → 추가 쿼리
   반복 후 Cloudflare/Qwen timeout (~24s → 500).

## 수정

### 공통 시스템 프롬프트 강화 (buildChatMessages)
- **환각 금지**: "도구가 반환하지 않은 수치·규칙·조문은 절대 지어내지 마"
- **LMS 정책 원칙**: search_wiki 결과만 근거로 사용, 일반 상법·회사법·민법 섞지 말 것.
  "LMS 는 시뮬레이션이라 실제 법과 규칙이 다름" 명시
- **wiki 미발견 시**: "공식 가이드에 명시돼 있지 않습니다" + 담당자 문의 안내
- **도구 호출 최소화**: 필요한 질문 하나만, 첫 결과로 충분하면 즉시 답변

### 응답 크기 / 체인 제한
- `maxToolHops`: 6 → **4**
- `context7_docs` 기본 tokens: 3000 → **1500**
- `fetch_url` 기본 max_chars: 6000 → **3500**

### dev_helper 프롬프트 재작성
- "도구 호출은 최대 2번" 명시
- 기본 모델 `qwen-reasoning medium` → **`qwen-chat`** (빠른 모드 기본)
- "첫 도구 결과로 즉시 답변, 추가 조회 금지"

## 예상 효과
- LMS 정책 질문의 환각 감소 (wiki-only 원칙)
- 긴 개발 질문에서 500 감소 (체인 4-hop + 모델 변경)
- 응답 시간 단축 (reasoning → chat 모드)
