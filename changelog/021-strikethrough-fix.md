# 021. 마크다운 취소선 오작동 수정: ~를 범위 표현에 안전하게

**날짜**: 2026-03-18
**태그**: `마크다운`, `버그수정`, `UX`

## 무엇을 했나요?

마크다운에서 `~` 한 개로 감싸면 취소선이 되는 문제를 수정했습니다. 이제 `~~` 두 개로 감싸야만 취소선이 됩니다.

## 왜 필요했나요?

한국어에서 `~`는 범위를 표현할 때 자주 씁니다:
- "3~5일 소요" → ~~의도치 않은 취소선~~
- "100~200만원" → 마찬가지

GFM(GitHub Flavored Markdown)은 `~한 개~`도 취소선으로 인식하는데, 이는 한국어 글쓰기에서 자주 문제를 일으킵니다.

## 어떻게 해결했나요?

`remark-gfm` 플러그인에 `singleTilde: false` 옵션을 추가했습니다.

```tsx
// Before
remarkPlugins={[remarkGfm]}

// After
remarkPlugins={[[remarkGfm, { singleTilde: false }]]}
```

이렇게 하면:
- `~텍스트~` → 그냥 텍스트 (범위 표현에 안전)
- `~~텍스트~~` → ~~취소선~~ (의도적 사용만)

## 마크다운에서 자주 겪는 비슷한 문제들

| 문법 | 의도된 용도 | 일상 글쓰기에서 충돌 |
|------|------------|---------------------|
| `~텍스트~` | 취소선 | 범위 표현 (3~5일) |
| `*텍스트*` | 이탤릭 | 강조 부호로 사용 |
| `_텍스트_` | 이탤릭 | 파일명의 언더스코어 |
| `#` | 제목 | 해시태그 |

**업계의 일반적인 해결 방법:**
1. **파서 옵션 조정** — 우리가 사용한 방법. 라이브러리가 제공하는 설정으로 동작 변경.
2. **전처리(preprocessing)** — 렌더링 전에 문제 패턴을 이스케이프 처리.
3. **커스텀 파서** — 직접 파서 플러그인을 작성하여 원하는 동작만 구현.

## 사용한 프롬프트

```
~의 경우 취소선 사용되는데, 범위를 표현할 때 의도와 다르게 취소선이 생기는 경우가 많아.
~~ 두번 연달아 쓰는 경우만 취소선으로 인식하도록 바꾸면 좋겠어.
```

## 배운 점

- **로케일 고려**: 영어권에서 만든 라이브러리의 기본 설정이 한국어 환경에서는 문제가 될 수 있습니다. 라이브러리 도입 시 현지화 이슈를 점검하세요.
- **옵션부터 확인**: 코드를 직접 수정하기 전에 라이브러리가 제공하는 설정 옵션을 먼저 확인하면 훨씬 깔끔하게 해결할 수 있습니다.
