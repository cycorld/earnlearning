# 137. 거래소 기업 카드가 붙어 보이던 문제 + 가격이 `NaN원/주` 로 뜨던 문제

**날짜**: 2026-06-29
**티켓**: #137
**타입**: 버그수정 / UX

## 무엇을 했나요?

거래소(`/exchange`) 탭의 **상장 기업 목록**에서 두 가지를 고쳤습니다.

- **A. 가격이 `NaN원/주` 로 보이던 문제**: 기업 카드의 주당 가격 자리에 숫자 대신 `NaN`(Not a Number, 계산 불가)이 표시됐습니다. 이제 **시가**(직전 체결가)가 제대로 나옵니다.
- **B. 카드가 세로로 딱 붙어 보이던 문제**: 기업 카드들 사이에 간격이 없어 답답했는데, 카드 사이 여백을 살렸습니다.

## 왜 필요했나요?

### A — 프론트가 받지도 않는 필드로 계산하고 있었다
거래소 목록 API(`GET /exchange/companies`)는 백엔드의 **`ListedCompany`** 모양으로 응답합니다 — `last_price`(시가), `market_cap`(시가총액) 등. 그런데 프론트는 이 응답을 엉뚱하게 **`Company`** 타입(`valuation` 필드를 가진)으로 간주하고 `valuation ÷ total_shares` 로 주당 가격을 계산했습니다. 응답에 `valuation` 이 **없으니** `undefined ÷ 숫자 = NaN` 이 됐습니다.

### B — inline 요소에는 세로 여백이 안 먹는다
카드 사이 간격은 Tailwind 의 `space-y-*`(= 형제 요소들에 위쪽 바깥여백을 줌)로 줍니다. 그런데 기업 카드는 **링크(`<Link>` → `<a>`)로 감싸** 있었고, `<a>` 는 기본이 **inline** 요소입니다. inline 요소에는 위/아래 margin 이 적용되지 않아 간격이 사라졌습니다. (#136 투자처 카드와 똑같은 함정입니다.)

## 어떻게 만들었나요?

### A
- 프론트에 실제 응답 모양인 `ListedCompany` 인터페이스를 정의하고, 주당 가격은 계산 없이 `company.last_price`(시가)를 그대로 씁니다.
- **시가 폴백 체인**(백엔드 `GetListedCompanies` SQL): 회사의 현재 가치는 **시가**(직전 체결가)인데, 아직 거래가 한 번도 없었다면 ① **마지막으로 펀딩된 투자 라운드의 주당 가격**으로, 그것도 없으면 ② **회사 가치 ÷ 총 주식수**로, 그래도 없으면 0 으로 떨어집니다. 거래·라운드가 아직 없는 갓 상장된 회사도 `0원` 대신 회사 가치를 보여줍니다.

### B
- 기업 카드를 감싼 `<Link>` 에 `className="block"` 추가 + 목록 컨테이너 간격을 `space-y-3` 로. inline 이던 앵커가 block 이 되면서 세로 간격이 정상 적용됩니다.

## 배운 점

- **API 응답 타입을 "진짜 오는 모양"으로 맞추자.** 타입을 대충 맞추면(`Company` 로 가정) 컴파일은 통과해도 런타임에 `undefined` 필드를 만나 `NaN` 같은 조용한 버그가 됩니다.
- **`NaN` 은 거의 항상 "없는 값으로 계산"의 신호다.** `undefined`/`null` 이 산술에 끼면 `NaN` 이 나옵니다. 가격·수량이 `NaN` 이면 입력 데이터부터 의심하세요.
- **CSS 간격 유틸(`space-y-*`)은 요소의 display 를 탄다.** 클릭 영역 때문에 카드를 링크로 감쌀 땐 `block`(또는 `flex`)으로 만들어야 합니다. (#136 과 동일 교훈 — 같은 패턴이 다른 페이지에 반복됨)
