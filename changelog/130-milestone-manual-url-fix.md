# 130. 평가지표 MVP 링크 직접 수정이 되돌아가는 버그 수정

**날짜**: 2026-06-12
**티켓**: #130
**타입**: 버그 수정

## 무엇을 했나요?

평가지표(/milestones) 페이지에서 학생이 잘못 자동 집계된 MVP1/2 링크를 "다시 제출"로 직접 고쳐도, 화면을 새로고침하면 원래의 잘못된 링크로 되돌아가는 버그를 고쳤습니다.

## 왜 생긴 버그인가요?

MVP 링크는 회사의 service_url이나 정부과제 응모 글에서 **자동으로 추출**됩니다(SyncAuto). 이 자동 동기화가 페이지를 조회할 때마다 실행되는데, 기존에는 "승인됨(approved)" 상태만 건너뛰고 나머지는 무조건 덮어썼습니다. 그래서 학생이 직접 고친 값(pending 상태)도 다음 조회 때 자동 추출 값으로 되돌아갔습니다 — 화면상으로는 "수정이 안 되는" 것처럼 보였습니다.

같은 원리로 또 하나의 버그도 발견했습니다: 교수님이 **반려(rejected)**한 자동 항목도 매 조회마다 같은 URL로 다시 저장되면서 상태가 "대기중"으로 리셋되고 교수님 코멘트가 지워졌습니다.

## 어떻게 고쳤나요?

자동 동기화(SyncAuto)의 건너뛰기 규칙을 확장했습니다:

1. **직접 제출(manual) 항목은 절대 덮어쓰지 않음** — 학생이 직접 입력한 값이 항상 우선합니다.
2. **URL과 출처가 그대로면 다시 저장하지 않음** — 반려 상태와 교수님 코멘트가 유지됩니다.
3. 승인된 항목 보호는 기존대로 유지. 회사 URL이 실제로 바뀌면 자동 갱신도 기존대로 동작합니다.

## 배운 점

- **"수정이 안 돼요"는 저장 실패가 아닐 수 있다**: 저장은 잘 되는데 다른 프로세스(자동 동기화)가 도로 덮어쓰는 경우, 사용자에게는 똑같이 "수정 안 됨"으로 보입니다. 데이터가 변하는 모든 경로를 추적해야 원인이 보입니다.
- **TDD로 버그 재현 먼저**: 고치기 전에 버그를 재현하는 테스트 4개를 먼저 작성해 실패를 확인하고(Red), 수정 후 통과(Green)시켰습니다. 이 테스트는 회귀 방지를 위해 계속 남습니다.
