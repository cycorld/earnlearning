# 139. 거래소 종목 상세 개편 — 체결 차트 + 거래이력 + 매수/매도 주문 UX

**날짜**: 2026-06-30
**티켓**: #139
**타입**: 기능추가 / UX / 버그수정

## 무엇을 했나요?

거래소에서 기업을 누르면 나오는 **종목 상세 화면**(`/exchange/:id`)을 "주식 앱처럼" 새로 기획했습니다.

1. **현재가 + 등락률**을 큼직하게 (상승=빨강, 하락=파랑 — 한국 증시 색 관습)
2. **가격 차트**: 체결가가 시간순으로 어떻게 움직였는지 라인+영역 그래프로 표시
3. **체결 내역(거래이력)**: 최근 거래를 `시각 · 가격 · 수량` 으로 나열, 직전 대비 오르면 빨강·내리면 파랑
4. **매수/매도 주문을 편하게**: 드롭다운 대신 `[매수][매도]` 탭, **호가를 누르면 그 가격이 주문 폼에 자동 입력**
5. 곁들여 **두 개의 숨어 있던 버그**도 고쳤습니다 (아래 "왜 필요했나요").

## 왜 필요했나요?

기존 상세 화면은 호가창과 주문 폼만 있어서 "지금 얼마인지, 어떻게 움직였는지"를 알 수 없었습니다. 거래이력과 차트가 있어야 매수·매도 판단을 할 수 있습니다.

작업하며 발견한 버그 2개:

- **호가창이 한 번도 뜬 적이 없었다.** 라우트는 `/exchange/orderbook/:companyId` 인데 백엔드 핸들러는 `companyId` 가 아닌 `id` 라는 이름으로 값을 꺼내고 있었습니다. 이름이 안 맞으니 빈 문자열 → 항상 `400` 에러 → 프론트는 에러를 조용히 무시 → 호가창이 `null` 로 안 보였습니다.
- **프론트가 호가 필드 이름을 잘못 알고 있었다.** 백엔드는 매도호가를 `asks`, 매수호가를 `bids` 로 주는데 프론트는 `sell_orders`/`buy_orders` 로 읽고 있었습니다. (위 400 에러에 가려 드러나지 않던 두 번째 버그)
- **목록 화면 "내 주문" 가격이 `NaN원`.** 주문 가격 필드 이름이 백엔드는 `price_per_share` 인데 프론트는 `price` 로 읽어 `undefined` → `NaN` 이 됐습니다.

## 어떻게 만들었나요?

### 백엔드 — 체결 내역 API 신설
- `GET /exchange/trades/:companyId?limit=50` 추가. `stock_trades` 테이블에서 해당 회사의 체결을 **최신순**으로 돌려줍니다. (차트와 거래이력의 데이터 원천)
- 계층별로: `Repository.GetCompanyTrades()` (SQL) → `ExchangeUseCase.GetCompanyTrades()` (회사가 상장됐는지 검증) → `ExchangeHandler.GetTrades()` (HTTP) → 라우터 등록.
- 차트는 **별도 API 없이** 이 체결 목록을 프론트에서 그대로 그립니다. (가격을 시간순으로 이으면 그게 차트)

### 백엔드 — 호가창 파라미터 버그 수정
- `GetOrderbook` 핸들러가 `intParam(c, "id")` → `intParam(c, "companyId")` 로. 라우트 파라미터 이름과 일치시켰습니다.

### 프론트 — 차트는 라이브러리 없이 SVG 로 직접
- 차트 라이브러리(recharts 등)가 프로젝트에 없어, 의존성을 늘리지 않고 **순수 SVG `<path>`** 로 라인+영역 그래프를 그렸습니다. 체결가의 최저~최고를 화면 높이에 맞게 정규화(normalize)해 좌표로 변환합니다.
- 선이 가로로 늘어나도 굵기가 일정하도록 `vectorEffect="non-scaling-stroke"` 를 사용했습니다.

### 프론트 — "편하게 쓸 수 있게"
- 주문 유형을 `Select` 드롭다운 → `Tabs` 토글로 (한 번 탭으로 전환).
- **호가 클릭 → 가격 자동 입력**: 매도호가를 누르면 "그 가격에 매수", 매수호가를 누르면 "그 가격에 매도" 로 폼이 채워집니다.
- 필드 이름을 백엔드 응답(`asks`/`bids`, `price_per_share`)에 맞춰 정정.

## 배운 점

- **라우트 파라미터 "이름"이 핸들러와 정확히 같아야 한다.** `:companyId` 로 선언했으면 `c.Param("companyId")` 로 꺼내야 합니다. 이름이 다르면 빈 값이 와도 컴파일은 통과해 — 조용히 깨집니다. (#137 의 `NaN` 과 같은 부류: "타입은 맞는데 값이 틀린" 버그)
- **에러를 조용히 삼키면(`catch {}`) 버그가 숨는다.** 호가창이 안 보였지만 화면은 멀쩡해 보여서 아무도 몰랐습니다.
- **차트에 꼭 무거운 라이브러리가 필요한 건 아니다.** 점을 좌표로 바꿔 잇는 것뿐 — 간단한 그래프는 SVG 한 줄(`<path>`)로 충분합니다.
- **백엔드 응답의 진짜 필드 이름을 보고 프론트 타입을 맞추자.** `asks` vs `sell_orders`, `price_per_share` vs `price` — 추측하면 런타임에 깨집니다.
