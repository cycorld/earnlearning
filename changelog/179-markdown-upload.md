# Markdown 문서도 과제 첨부파일로 올리기

## 무엇을 했나요?

에디터의 파일 선택기에 `.md` 형식을 추가하고, 서버가 Markdown 파일을 안전하게 저장하도록 업로드 검증을 보완했습니다. PDF처럼 기존에 지원하던 파일도 계속 업로드되는지 회귀 테스트를 추가했습니다.

## 왜 필요한가요?

PRD와 SPEC은 Markdown으로 작성하면 제목, 목록, 코드 같은 구조를 간결하게 표현할 수 있습니다. 이전에는 문서를 작성해도 업로드 선택기와 서버 검증이 Markdown을 허용하지 않아 별도 형식으로 변환해야 했습니다.

## 어떻게 만들었나요?

먼저 Markdown 업로드가 실패하는 통합 테스트를 작성했습니다. 그 다음 `.md` 확장자와 브라우저별 MIME 형식(`text/markdown`, `text/plain`, `application/octet-stream`)만 제한적으로 허용했습니다. 파일명에는 경로 이동 문자나 제어 문자가 없는지 검사하고, 선언된 크기뿐 아니라 실제 복사한 바이트 수도 10MB 이하인지 확인합니다.

## 사용한 프롬프트

> 격리 worktree의 Markdown 업로드 구현을 완성하고 테스트하라

## 배운 점

파일 선택기의 `accept` 속성은 사용자 편의 기능일 뿐 보안 검증이 아닙니다. 서버가 확장자, MIME 형식, 실제 크기를 함께 확인해야 하며, 새 형식을 추가할 때는 기존 형식의 회귀 테스트도 필요합니다.
