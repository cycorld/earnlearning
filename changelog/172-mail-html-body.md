# 빈 본문의 정체 — HTML 전용 메일과 샌드박스 iframe

## 무엇을 했나요?

Anthropic 로그인 링크 같은 실제 서비스 메일이 수신함에서 **본문이 텅 비어** 보이던 문제를 고쳤습니다. 이제 HTML 메일이 버튼·스타일까지 제대로 렌더링됩니다. 목록 미리보기(스니펫)도 태그를 벗겨 채워줍니다.

## 왜 그랬나요?

처음 메일함을 만들 때 XSS(악성 스크립트 주입)를 막으려고 **텍스트 파트만** 화면에 그렸습니다. 그런데 실제 세상의 트랜잭션 메일 상당수는 텍스트 파트 없이 **HTML로만** 옵니다. 저장은 되고 있었는데 그릴 방법이 없었던 것.

## 어떻게 고쳤나요? — 보안을 포기하지 않고

`dangerouslySetInnerHTML`로 우리 페이지에 직접 주입하는 대신, **샌드박스 iframe**에 격리해서 렌더링합니다 (Gmail과 같은 계열의 접근):

- `sandbox` 속성에 `allow-scripts`를 **주지 않아** 메일 안의 자바스크립트는 아예 실행 불가
- 링크는 `<base target="_blank">` + `allow-popups`로 새 탭에서만 열림
- 검증: 테스트 메일에 `<script>alert(1)</script>`를 심어 보냈고 — 실행되지 않음을 실화면으로 확인

스니펫은 백엔드에서 텍스트 파트가 비어 있을 때만 HTML 태그를 제거한 텍스트로 폴백합니다 (`StripHTMLTags`).

## 배운 점

- "안전하게 안 그린다"는 임시방편이지 해법이 아닙니다. 실사용 첫날 실제 메일(Anthropic 로그인 링크)이 바로 구멍을 보여줬습니다.
- 신뢰할 수 없는 HTML을 다루는 표준 답은 격리(iframe sandbox)입니다. 스크립트를 차단한 채 같은 출처 읽기만 허용하면 높이 자동 조절도 안전하게 됩니다.
