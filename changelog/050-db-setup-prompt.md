# 050. DB 생성 시 바이브코딩용 연동 프롬프트 제공

> **날짜**: 2026-04-12
> **태그**: `feat`, `PostgreSQL`, `학생DB`, `UX`

## 무엇을 했나요?

학생이 프로필에서 PostgreSQL 데이터베이스를 생성하면, 접속 정보 아래에 **"DB 연동 프롬프트"** 섹션이 추가됩니다. 이 프롬프트를 복사해서 바이브코딩 도구(Cursor, Bolt, Claude 등)에 붙여넣으면 DB 연동 코드가 자동으로 생성됩니다.

## 왜 필요했나요?

학생들이 DB를 만든 후 "이걸 내 프로젝트에 어떻게 연결하지?"라는 질문을 자주 했습니다. HOST, PORT, DATABASE 같은 정보를 직접 프롬프트로 작성하는 것은 초보자에게 어려운 일이라, 원클릭으로 복사 가능한 완성된 프롬프트를 제공하기로 했습니다.

## 어떻게 만들었나요?

1. `CredentialsDialog` (DB 생성/비밀번호 재발급 시 표시되는 다이얼로그) 안에 `DbSetupPromptSection` 컴포넌트를 추가
2. 토글 방식으로 기본 숨김 → 클릭 시 펼침 (접속 정보가 이미 길어서 UI 과부하 방지)
3. `buildDbSetupPrompt()` 함수가 사용자의 실제 접속 정보를 포함한 마크다운 프롬프트를 생성
4. 기존 `CopyBlock` 컴포넌트를 재활용하여 원클릭 복사 지원

## 프롬프트에 포함되는 내용

- 접속 정보 (Host, Port, Database, Username, Password, Connection URL)
- 연동 요구사항 (환경변수 설정, 연결 테스트 코드)
- 참고사항 (PgBouncer 포트 안내, ORM/SQL 자유 선택)

## 배운 점

- **사용자의 다음 행동을 예측하라**: DB를 만든 직후 학생이 할 일은 "연동"입니다. 그 다음 단계를 미리 안내하면 이탈이 줄어듭니다.
- **토글 UI**: 정보량이 많을 때는 기본 숨김 + 토글로 제공하면 핵심 정보(비밀번호)에 집중할 수 있습니다.
