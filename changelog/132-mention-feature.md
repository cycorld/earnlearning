# 132. 게시글/댓글 @멘션 기능

**날짜**: 2026-06-13
**티켓**: #132
**타입**: 신규 기능

## 무엇을 했나요?

게시글이나 댓글을 쓸 때 `@`를 입력하면 **유저 자동완성 드롭다운**이 떠서 동료를 멘션할 수 있습니다. 멘션된 사람에게는 알림이 가고, 알림 페이지의 **"멘션" 탭**에서 나를 멘션한 글만 모아볼 수 있습니다. 댓글에서 멘션됐다면 알림 클릭 시 게시글로 이동한 뒤 **해당 댓글 위치로 자동 스크롤**됩니다.

## 왜 필요했나요?

수강생끼리 게시글에서 서로를 불러 협업할 방법이 없었습니다. 이름을 텍스트로 적어도 상대방은 알 수 없죠. 특히 한글 이름은 **동명이인**이 있어서(실제로 존재합니다) 이름만으로는 누구를 가리키는지 모호합니다.

## 어떻게 만들었나요?

- **동명이인 문제 → 자동완성 강제**: 자유 텍스트 `@이름`이 아니라, 드롭다운에서 선택하면 `@[이름](user:ID)` 형태로 **user ID가 확정 저장**됩니다. 드롭다운에는 이름 옆에 **전공·학번**을 함께 보여줘서 같은 이름이라도 구분할 수 있습니다.
- **Backend**:
  - `GET /api/users/search?q=` — 승인된 유저만 이름/학번 부분일치 검색.
  - 게시글/댓글 작성 시 본문에서 멘션 마크업을 정규식으로 파싱 → 새 알림 타입 `mention` 발송.
  - 댓글 멘션은 `notifications` 테이블에 새로 추가한 `anchor` 컬럼에 `comment-<id>`를 저장 → 프론트가 `/post/3#comment-12`처럼 이동.
  - 글 작성자가 댓글에서 멘션되면 기존 `new_comment` 알림은 생략(중복 방지).
- **Frontend**: `MarkdownEditor`에 자동완성 훅을 한 번만 통합해 피드 작성/수정/댓글 모두 적용. 본문 렌더 시 멘션은 프로필 링크(`@이름`)로 하이라이트.
- **TDD**: 통합 테스트 16건 먼저 작성 — 검색(동명이인·미승인 제외·전공 포함), 멘션 알림(셀프 멘션 제외·anchor·중복 방지·회귀), `?type=mention` 필터.

## 배운 점

- **알림에 "어디로 갈지"가 두 단계일 수 있다**: 페이지(`/post/3`)까지는 기존 reference로 충분하지만, 페이지 안의 위치(12번 댓글)는 별도 정보가 필요합니다. URL fragment(`#comment-12`) 관례를 그대로 DB에 저장하니 프론트 처리도 단순해졌습니다.
- **같은 사건에 알림 두 개는 스팸**: 댓글 하나로 mention + new_comment가 동시에 가면 받는 사람은 같은 일을 두 번 통지받습니다. 더 구체적인 알림(mention)이 덜 구체적인 것(new_comment)을 대체하게 했습니다.
