# 138. 평가지표 매트릭스 — 긴 에세이 평가 시 모달이 스크롤 안 되던 문제

**날짜**: 2026-06-30
**티켓**: #138
**타입**: 버그수정 / UX

## 무엇을 했나요?

관리자 > **평가지표 매트릭스**(`/admin/milestones`)에서 학생 마일스톤(에세이·회고 등)을 평가할 때 뜨는 **평가 모달(ReviewDialog)** 을 고쳤습니다.

- 제출 내용이 길면 모달이 화면(뷰포트)보다 커지는데도 **스크롤이 안 됐습니다**. 그 결과 맨 아래 **승인/반려 버튼이 화면 밖으로 밀려 클릭할 수 없었습니다**.
- 이제 모달 높이가 화면의 90%로 제한되고, **제출 내용 영역만 스크롤**됩니다. **헤더(학생명·항목)와 하단 코멘트/승인·반려 버튼은 항상 고정**되어 보입니다.

## 왜 필요했나요?

모달은 오버레이 안에서 `items-center`(세로 가운데 정렬)로 떠 있었고, 내용 카드에 **높이 제한도 스크롤 설정도 없었습니다**. 내용이 길어지면 카드가 위아래로 동시에 커지면서 **화면 위·아래로 잘려 나갔고**, 잘린 부분(하단 버튼)에는 접근할 방법이 없었습니다. 긴 에세이일수록 평가가 가장 필요한데, 정작 평가 버튼을 못 누르는 상황이었습니다.

## 어떻게 만들었나요?

CSS(Tailwind) 레이아웃만 바꿨고 로직은 건드리지 않았습니다 (`AdminMilestonesPage.tsx` 의 `ReviewDialog`).

- **카드 높이 제한**: 내용 카드에 `max-h-[90vh]` — 아무리 길어도 화면의 90%를 넘지 않습니다. (카드 컴포넌트는 이미 `flex flex-col overflow-hidden` 이라 세로 박스 구조는 그대로 활용.)
- **3단 세로 박스로 분리**: 카드 내부를 `flex flex-col` 로 만들어 **헤더 / 본문 / 푸터** 3층 구조로.
  - 헤더(학생·항목·닫기)와 푸터(코멘트 + 승인/반려)는 `flex-shrink-0` — **절대 줄어들거나 밀려나지 않음**.
  - 가운데 제출 내용 영역만 `flex-1 min-h-0 overflow-y-auto` — **남는 공간을 채우고, 넘치면 자기 안에서 스크롤**.
- 핵심은 `min-h-0`: flex 항목은 기본적으로 내용보다 작게 줄지 않아 스크롤이 안 생깁니다. `min-h-0` 을 줘야 자식이 부모보다 작아질 수 있고 그제서야 `overflow-y-auto` 가 동작합니다.

### 검증
- 프론트 빌드·타입체크 통과, 백엔드 스모크 24/24 통과.
- 짧은 내용 → 모달이 내용만큼만 작게(중앙), 긴 내용 → 90vh 로 고정 + 본문만 스크롤, 버튼 항상 노출 — browse-agent(헤드리스)로 시각 확인.

## 배운 점

- **모달에는 높이 상한과 스크롤을 함께 줘야 한다.** 둘 중 하나만 있으면 내용이 길 때 버튼이 화면 밖으로 사라지는 "보이지 않는 막다른 길"이 생깁니다.
- **`overflow-y-auto` 가 안 먹으면 부모 flex 사슬의 `min-h-0` 을 의심하라.** flex 항목의 기본 `min-height: auto` 때문에 스크롤이 안 생기는 함정은 매우 흔합니다.
- **"가장 길 때"가 가장 중요한 순간이다.** 긴 에세이일수록 꼼꼼히 평가해야 하는데 그때 버튼을 못 누르면 기능 자체가 무력화됩니다. 엣지 케이스가 곧 핵심 케이스일 수 있습니다.
