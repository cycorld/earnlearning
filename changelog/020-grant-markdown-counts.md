# 020. 정부과제 지원서 마크다운 + 지원/승인 현황 표시

**날짜**: 2026-03-18
**태그**: `정부과제`, `마크다운`, `UX`

## 무엇을 했나요?

1. **지원서 마크다운 뷰어**: 정부과제 지원서(proposal)가 마크다운으로 렌더링되도록 변경
2. **지원자수/승인자수 표시**: 정부과제 목록에서 각 과제의 지원 현황을 한눈에 확인 가능

## 왜 필요했나요?

- 지원서를 작성할 때는 마크다운 에디터를 사용하지만, 관리자가 볼 때는 plain text로 보여서 포맷이 깨졌습니다
- 과제 목록에서 지원자가 몇 명인지, 승인된 인원이 몇 명인지 확인할 수 없었습니다

## 어떻게 만들었나요?

### 지원서 마크다운
- GrantDetailPage에서 `<p>{app.proposal}</p>` → `<MarkdownContent content={app.proposal} />` 로 변경
- 피드 타임라인과 동일한 마크다운 렌더러 사용

### 지원/승인 현황
- **Backend**: `approved_count` 서브쿼리 추가 (grant_applications에서 status='approved' 카운트)
- **Frontend**: 목록에 지원 N명 / 승인 N명 배지 표시

## 사용한 프롬프트

```
정부과제 지원서를 markdown viewer 가 가능하도록 해줘. 타임라인에서 잘 표현 되듯이.
그리고 정부과제 메뉴에서 지원자수와 승인자수를 표시해줘.
```

## 배운 점

- **일관된 렌더링**: 입력과 출력에서 같은 포맷을 사용해야 합니다. 마크다운 에디터로 입력받으면서 plain text로 보여주면 사용자 경험이 나빠집니다.
- **서브쿼리 활용**: SQL에서 `(SELECT COUNT(*) FROM ... WHERE ...)` 서브쿼리로 관련 집계를 한 번의 쿼리로 가져올 수 있습니다.
