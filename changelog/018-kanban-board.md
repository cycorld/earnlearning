# 칸반보드 태스크 관리: 개발 작업을 시각적으로 관리하기

**날짜**: 2026-03-18
**태그**: `관리자`, `칸반보드`, `태스크`, `프로젝트관리`

## 무엇을 했나요?

관리자 페이지에 **칸반보드 형태의 태스크 관리** 기능을 추가했습니다. Backlog → To Do → In Progress → Done 4개 컬럼으로 개발 작업을 시각적으로 관리할 수 있습니다.

## 왜 필요했나요?

개발할 기능, 버그 수정, 콘텐츠 작업 등을 체계적으로 추적하고 싶었습니다. 별도의 프로젝트 관리 도구 없이 LMS 안에서 바로 관리할 수 있습니다.

## 어떻게 만들었나요?

### 백엔드
`kanban_tasks` 테이블을 만들고 CRUD API를 구현했습니다. 각 태스크는 제목, 설명, 우선순위(high/medium/low), 타입(feat/fix/chore/content), 상태(backlog/todo/in-progress/done)를 가집니다.

### 프론트엔드
4개 컬럼의 칸반보드 UI를 만들었습니다. 각 카드에 좌우 화살표 버튼으로 상태를 이동할 수 있고, 카드를 클릭하면 수정 다이얼로그가 열립니다.

## 사용한 프롬프트

```
태스크 폴더 만들어서 칸반보드처럼 관리하고 싶어. 관리자 페이지에 개설해줘.
```

## 배운 점

칸반보드는 단순하지만 강력한 프로젝트 관리 도구입니다. 4개 상태만으로도 작업 흐름을 명확하게 파악할 수 있습니다.
