# 123. 보상 금액 상향 + 게시글 작성 보상 신설

**날짜**: 2026-06-12
**티켓**: #123
**타입**: 신규 기능 / 개선

## 무엇을 했나요?

커뮤니티 참여 보상을 대폭 상향하고, **게시글을 작성하기만 해도 보상**을 받는 시스템을 새로 추가했습니다.

| 행동 | 변경 전 | 변경 후 | 수령자 |
|------|--------|--------|--------|
| 게시글 작성 | 없음 | **+10,000원** (신규) | 작성자 본인 |
| 게시글 삭제 | 없음 | **−10,000원 회수** (신규) | 작성자 본인 |
| 좋아요 | +10원 | **+500원** | 글쓴이 |
| 댓글 | +100원 | **+1,000원** | 글쓴이 |

## 왜 필요했나요?

기존 보상(좋아요 10원·댓글 100원)은 금액이 너무 작아 학생들이 체감하기 어려웠습니다. 보상을 현실감 있게 키우고, 무엇보다 **글을 쓰는 행동 자체**에 가장 큰 보상(1만원)을 부여해 "일단 써보는" 진입 장벽을 낮추려는 목적입니다. 콘텐츠가 많아질수록 좋아요·댓글도 늘어나는 선순환을 노립니다.

## 어떻게 만들었나요?

### 보상 규칙
- **게시글 작성**: 작성에 성공하면 작성자 지갑에 10,000원이 입금됩니다. 단 **과제 게시글**과 **시스템 자동 게시글**은 일반 글 작성 경로(`CreatePost`)를 타지 않으므로 보상에서 제외됩니다.
- **게시글 삭제 = 보상 회수**: 글을 지우면 작성 보상 10,000원을 회수합니다. 어뷰징(작성→삭제→재작성 반복)을 막기 위해, **관리자가 삭제한 경우에도** 작성자에게서 회수합니다.
- **좋아요/댓글**: 금액만 500원/1,000원으로 올렸고, 기존 규칙(자기 글 제외, 취소·삭제 시 회수)은 그대로입니다.

### Backend 변경
1. `post_reward` 트랜잭션 타입을 추가했습니다.
2. 보상 금액 상수를 한곳에서 관리합니다: `PostRewardAmount=10000`, `LikeRewardAmount=500`, `CommentRewardAmount=1000`.
3. `CreatePost`에 작성 보상 지급, `DeletePost`에 회수 로직을 추가했습니다.

### Frontend 변경
- 게시글 작성 시 "작성 보상 +10,000원" 토스트 알림을 표시합니다.
- 댓글 보상 토스트 금액을 1,000원으로 갱신했습니다. (좋아요 토스트는 서버 응답값을 그대로 쓰므로 자동 반영됩니다.)

### TDD로 개발
기존 좋아요/댓글 회귀 테스트의 기대값을 새 금액(500/1,000)으로 갱신하고, 게시글 작성 보상에 대한 통합 테스트를 추가했습니다:
- 게시글 작성 → 작성자 +10,000원 + `post_reward` 거래 기록
- 본인 글 삭제 → 10,000원 회수
- 관리자가 학생 글 삭제 → 작성자에게서 10,000원 회수
- 과제 생성 → 작성 보상 없음(제외 확인)

## 배운 점
- **보상은 "하길 원하는 행동"에 가장 크게**: 우리가 가장 원하는 행동(글쓰기)에 가장 큰 보상을 두면 행동 유도가 명확해집니다.
- **회수까지 설계해야 어뷰징을 막는다**: "지급" 규칙만 만들면 작성→삭제 반복으로 무한 파밍이 가능합니다. 반대 방향(회수)을, 그것도 관리자 삭제까지 포함해 함께 설계해야 합니다.

## 알려진 한계
- 작성자가 보상금을 먼저 다 써버린 뒤 글을 삭제하면, 회수 시 잔액이 부족해 회수가 실패할 수 있습니다(기존 좋아요/댓글 회수와 동일한 특성). 강제 회수(잔액 음수 허용)는 경제 전반에 영향이 있어 후속 티켓에서 검토합니다.
- 글 작성 횟수 자체에는 상한(cap)이 없습니다. 다수의 글로 보상을 모으는 것은 막지 않습니다.
