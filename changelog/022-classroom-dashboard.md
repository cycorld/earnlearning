# 022. 강의실 학생 대시보드: 학생 현황을 한눈에

**날짜**: 2026-03-19
**태그**: `관리자`, `대시보드`, `강의실`, `UX`

## 무엇을 했나요?

관리자 페이지의 강의실 관리에 **강의실 상세 대시보드**를 추가했습니다. 각 학생의 경제 활동 현황을 한눈에 확인할 수 있습니다.

## 왜 필요했나요?

교수가 학생들의 수업 참여도와 게임 내 경제 활동 상태를 빠르게 파악할 방법이 없었습니다. 학생 한 명씩 프로필을 확인해야 했습니다.

## 어떻게 만들었나요?

### Backend
- `GET /admin/classrooms/:id/dashboard` API 추가
- SQL 한 방으로 멤버별 지갑, 회사, 대출, 게시글 수를 집계
- 서브쿼리로 총 자산(현금 + 주식 - 부채) 계산

### Frontend
- `AdminClassroomDetailPage.tsx` 신규 생성
- 요약 카드 (학생 수, 평균 자산, 총 회사 수, 총 부채)
- 학생 리스트: 이름/학번 검색, 정렬 (자산, 현금, 회사, 게시글, 부채, 이름)
- 학생 카드에 총자산, 현금, 회사명, 부채, 게시글 수 표시
- 클릭하면 학생 프로필로 이동

## 사용한 프롬프트

```
관리자페이지->강의실관리->강의실 상세페이지를 만들어서 등록된 학생들의 상태를 볼 수 있는 대시보드를 만들어줘.
```

## 배운 점

- **SQL 서브쿼리 활용**: 여러 테이블의 집계를 서브쿼리로 한 번에 가져오면 N+1 문제를 피할 수 있습니다.
- **정렬/필터는 프론트에서**: 데이터 양이 한 반 분량(~30명)이면 서버 정렬보다 클라이언트 정렬이 더 빠르고 UX가 좋습니다.
