# 자동 피드 포스팅 & 알림 보강: 활동이 피드에 자동으로 올라가게

**날짜**: 2026-03-16
**태그**: `피드`, `자동포스팅`, `알림`, `이메일`, `거래소`, `투자`, `외주`, `정부과제`

## 무엇을 했나요?

각 도메인의 주요 활동이 발생하면 해당 피드 채널에 **자동으로 포스팅**되도록 했습니다. 또한 누락된 알림을 보강하여 상대방에게 푸시+이메일 알림이 가도록 했습니다.

## 자동 피드 포스팅

| 활동 | 피드 채널 | 포스팅 내용 |
|------|-----------|-------------|
| 정부과제 생성 | 과제 | 공고 내용, 보상, 모집 인원 |
| 외주 의뢰 등록 | 외주마켓 | 의뢰 내용, 예산, 필요 스킬 |
| 외주 완료 보고 | 외주마켓 | 완료 보고서 |
| 투자 라운드 오픈 | 투자라운지 | 목표 금액, 지분, 주당 가격 |
| 주식 거래 체결 | 거래소 | 종목, 수량, 체결 가격 |

## 알림 보강

| 상황 | 누구에게 | 기존 | 변경 |
|------|----------|------|------|
| 주식 매수 체결 | 매수자 | ❌ | ✅ 푸시+이메일 |
| 주식 매도 체결 | 매도자 | ❌ | ✅ 푸시+이메일 |
| 투자 완료 | 투자자 본인 | ❌ | ✅ 푸시+이메일 |
| 과제 채점 | 학생 | ❌ | ✅ 푸시+이메일 |
| 배당금 수령 | 주주 | DB만 저장 | ✅ 푸시+이메일 |
| 투자 유치 | 회사 대표 | DB만 저장 | ✅ 푸시+이메일 |

## 어떻게 만들었나요?

### AutoPoster 공통 헬퍼

각 도메인에서 피드에 글을 쓰는 코드가 중복되지 않도록 `AutoPoster`라는 공통 헬퍼를 만들었습니다.

```go
type AutoPoster struct { db *sql.DB }

// 사용자의 클래스룸에서 해당 slug 채널을 찾아 자동 포스팅
func (ap *AutoPoster) PostToChannel(slug string, authorID int, content string, tags []string)

// 관리자 계정으로 자동 포스팅 (시스템 이벤트용)
func (ap *AutoPoster) PostToChannelAsAdmin(slug string, content string, tags []string)
```

### 알림 경로 통합

기존 Investment/Exchange는 DB에 직접 INSERT하거나 알림을 아예 안 보내고 있었습니다. 이를 모두 `NotificationUseCase.CreateNotification()`을 경유하도록 변경하여 **WebSocket + Push + Email** 3채널 알림이 동시에 작동하게 했습니다.

## 사용한 프롬프트

```
정부과제에 올라오는 활동들은 피드-과제에 자동 포스팅 되면 좋겠어.
그리고 마켓->외주마켓, 투자->투자라운지, 거래소->거래소 피드에 자동 포스팅 되도록 해줘.
그리고 서로 인터렉션이 있을 때 상대방에게 노티가 가고 있는지 다 확인해줘. 이메일 알림도 마찬가지임.
```

## 배운 점

1. **공통 헬퍼 패턴**: 여러 도메인에서 같은 작업(피드 포스팅)을 할 때, 공통 헬퍼를 만들면 코드 중복을 줄이고 일관성을 유지할 수 있습니다.
2. **Setter 주입 패턴**: 순환 의존성을 피하기 위해 `SetNotificationUseCase()` setter를 사용했습니다. 생성 시점에는 notifUC가 없지만, 모든 use case 생성 후 주입합니다.
3. **알림 경로 통합의 중요성**: DB 직접 INSERT로 알림을 만들면 Push/Email이 안 갑니다. 반드시 NotificationUseCase를 경유해야 3채널 알림이 작동합니다.
