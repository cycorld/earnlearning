# 035. 게시글 삭제 기능 + 삭제 확인 다이얼로그

> **날짜**: 2026-04-03
> **태그**: `피드`, `UX`, `안전장치`

## 무엇을 했나요?

게시글 삭제 기능을 추가하고, 실수 방지를 위해 "삭제" 텍스트를 직접 입력해야 삭제되는 확인 다이얼로그를 적용했습니다.

## 어떻게 만들었나요?

- 연필 아이콘 클릭 시 "수정"과 "삭제" 두 옵션이 있는 드롭다운 메뉴 표시
- 삭제 선택 시 다이얼로그가 열리고, "삭제"를 정확히 입력해야 삭제 버튼이 활성화
- 정부과제 지원서 삭제에도 동일한 확인 다이얼로그 적용
- 댓글 아바타가 댓글 작성자 이름 옆 상단에 정렬되도록 수정

## 배운 점

- 삭제처럼 되돌릴 수 없는 작업에는 단순 confirm()보다 텍스트 입력 확인이 더 안전합니다. GitHub의 리포지토리 삭제와 같은 패턴입니다.
