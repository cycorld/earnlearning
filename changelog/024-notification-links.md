# 024. 알림 클릭 시 관련 페이지 이동

**날짜**: 2026-03-19
**태그**: `알림`, `UX`, `네비게이션`

## 무엇을 했나요?

알림 목록에서 알림을 클릭하면 해당 알림과 관련된 페이지로 이동하도록 누락된 매핑을 추가했습니다.

## 왜 필요했나요?

알림을 클릭해도 이동되지 않는 경우가 많았습니다. 백엔드에서 알림 생성 시 `reference_type`을 설정하지만, 프론트엔드에서 이 값을 URL로 변환하는 매핑이 일부만 구현되어 있었습니다.

## 어떻게 만들었나요?

`getReferencePath()` 함수에 누락된 reference_type 매핑을 추가했습니다:

| reference_type | 이동 페이지 |
|---|---|
| `freelance_job` | `/market/:id` (외주 마켓) |
| `grant` | `/grant/:id` (정부과제) |
| `assignment`, `submission` | `/feed` (피드) |
| `dividend` | `/invest` (투자) |
| `admin_transfer` | `/wallet` (지갑) |

아이콘도 알림 유형별로 더 세분화했습니다 (정부과제, 외주 마켓 등).

## 사용한 프롬프트

```
알림리스트에서 해당 알림을 클릭하면 관련 페이지로 이동 되도록 해줘.
```

## 배운 점

- **매핑 테이블 패턴**: 백엔드와 프론트엔드 사이에 문자열 기반 타입 매핑이 있을 때, 새 기능 추가 시 양쪽 모두 업데이트해야 합니다. 한쪽만 수정하면 조용히 실패합니다.
