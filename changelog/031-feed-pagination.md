# 031. 피드 페이지네이션 추가

> **날짜**: 2026-04-02
> **태그**: `UX`, `피드`, `페이지네이션`

## 무엇을 했나요?

피드(홈) 페이지에 이전/다음 페이지네이션 버튼을 추가했습니다.

## 왜 필요했나요?

기존에는 `page=1&limit=20`으로 고정되어 20개 이후의 게시글을 볼 수 없었습니다.

## 어떻게 만들었나요?

- `fetchPosts`에 `page` 파라미터를 추가하고 API 응답의 `pagination` 정보를 state에 반영
- 피드 하단에 "이전 / 1 of N / 다음" 페이지네이션 UI 추가
- 채널이나 태그 필터가 변경되면 자동으로 1페이지로 리셋

## 배운 점

- 백엔드 API가 이미 `PaginatedData` 형태(page, limit, total, total_pages)로 응답하고 있었으므로, 프론트엔드에서 이를 활용하기만 하면 됐습니다. API 설계 시 pagination을 처음부터 넣어두면 나중에 편합니다.
