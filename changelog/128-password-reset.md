# 128. 이메일 기반 비밀번호 찾기

**날짜**: 2026-06-12
**티켓**: #128
**타입**: 신규 기능

## 무엇을 했나요?

비밀번호를 잊은 사용자가 스스로 복구할 수 있는 **비밀번호 찾기** 기능을 만들었습니다.

1. 로그인 페이지의 "비밀번호를 잊으셨나요?" 링크 → 이메일 입력
2. 가입한 이메일로 **재설정 링크**(1시간 유효, 1회용)가 발송됩니다
3. 링크를 누르면 새 비밀번호를 설정하는 페이지가 열립니다

## 왜 필요했나요?

지금까지는 비밀번호를 잊으면 복구할 방법이 전혀 없었습니다. 관리자조차 비밀번호를 다시 설정해주는 API가 없어서, 계정을 잃어버리면 그대로 막히는 상황이었습니다.

## 어떻게 만들었나요?

### 백엔드 (Go)

- **`password_reset_tokens` 테이블** — 토큰 원본이 아니라 **SHA-256 해시**만 저장합니다. DB가 유출되어도 토큰을 역산할 수 없습니다.
- **`POST /api/auth/forgot-password`** — 토큰을 만들어 AWS SES로 이메일을 보냅니다. 중요한 보안 포인트: **이메일이 등록되어 있는지 여부를 응답에서 알 수 없게** 항상 같은 성공 메시지를 돌려줍니다 (계정 존재 탐색 공격 방지).
- **`POST /api/auth/reset-password`** — 토큰 검증 후 bcrypt로 새 비밀번호를 해싱해 저장합니다. 토큰은 한 번 쓰면 무효화되고, 새 요청을 하면 이전 토큰도 무효화됩니다.
- SES가 설정되지 않은 개발 환경에서는 메일 대신 서버 로그에 재설정 URL을 출력합니다.

### 프론트엔드 (React)

- `/forgot-password` — 이메일 입력 페이지
- `/reset-password?token=...` — 새 비밀번호 설정 페이지
- 로그인 페이지에 "비밀번호를 잊으셨나요?" 링크 추가

### 테스트 (TDD)

기능 코드를 작성하기 **전에** 통합 테스트 7개를 먼저 작성했습니다 (Red → Green):

- 전체 플로우 (요청 → 메일 → 재설정 → 새 비밀번호로 로그인)
- 토큰 재사용 차단, 새 요청 시 이전 토큰 무효화
- 미등록 이메일에도 성공 응답 (존재 비노출)
- 위조/만료 토큰 거부, 8자 미만 비밀번호 거부

테스트에서는 실제 메일을 보내는 대신 **가짜(fake) 이메일 발송기**를 주입해 메일 내용을 가로채 토큰을 추출합니다.

## 배운 점

- **토큰은 해시로 저장**: 세션 토큰이든 재설정 토큰이든, 서버가 다시 원본을 알 필요가 없는 값은 해시만 저장하는 것이 안전합니다.
- **응답 균일화**: "이 이메일은 가입되지 않았습니다" 같은 친절한 에러가 공격자에게는 회원 명단을 캐는 도구가 됩니다.
- **의존성 주입**: 이메일 발송기를 인터페이스로 분리하면 테스트에서 진짜 메일 없이 전체 플로우를 검증할 수 있습니다.
