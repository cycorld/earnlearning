# 019. 게시글 수정 기능 + 금액 포맷팅 통합

**날짜**: 2026-03-18
**태그**: `피드`, `게시글수정`, `포맷팅`, `리팩토링`, `UX`

## 무엇을 했나요?

두 가지 기능을 동시에 구현했습니다:

1. **게시글 수정 기능**: 피드에서 내가 작성한 글을 수정할 수 있게 됨
2. **금액 세 자리 콤마 포맷**: 서비스 전체에서 돈 표시를 통일

## 왜 필요했나요?

- 게시글을 한 번 쓰면 오타나 내용 수정이 불가능해서 불편했습니다
- 금액 표시 함수(`formatMoney`)가 22개 파일에 각각 중복 정의되어 있어 유지보수가 어려웠습니다

## 어떻게 만들었나요?

### 게시글 수정

**Backend (Go)**:
- `PUT /api/posts/:id` API 엔드포인트 추가
- 본인이 쓴 글만 수정 가능 (admin은 모든 글 수정 가능)
- 수정 시에도 `#태그` 자동 추출 로직 동일하게 적용

**Frontend (React)**:
- 본인 글에만 연필(수정) 아이콘 표시
- 클릭하면 수정 다이얼로그가 열리고, 기존 내용과 태그가 채워짐
- 수정 완료 시 목록에 즉시 반영 (낙관적 업데이트)

### 금액 포맷 통합

- `frontend/src/lib/utils.ts`에 공통 `formatMoney()` 함수 생성
- 22개 파일에서 중복 함수를 삭제하고 공통 import로 교체
- `1500000` → `1,500,000원` 형태로 표시

## 사용한 프롬프트

```
내가 쓴글 수정 기능이 필요해. 그리고 모든 돈은 세자리마다 , 표시 되도록 해줘.
```

## 배운 점

- **DRY 원칙**: 같은 함수가 22곳에 복사되어 있으면 버그 수정 시 22곳을 모두 고쳐야 합니다. 공통 유틸리티로 추출하면 한 곳만 수정하면 됩니다.
- **권한 체크는 백엔드에서**: 프론트에서 버튼을 숨기더라도, API에서도 반드시 "본인 글인지" 검증해야 합니다.
- **Intl.NumberFormat**: JavaScript 내장 API로 숫자 포맷팅을 간편하게 할 수 있습니다. `'ko-KR'` 로케일을 지정하면 한국식 3자리 콤마가 자동 적용됩니다.
