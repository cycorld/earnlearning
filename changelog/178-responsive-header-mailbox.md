# #178 PC 본문 폭 및 모바일 헤더·메일함 선택 UX 개선

## 무엇을 바꿨나요?

- 좁은 모바일 헤더를 로고·알림 영역과 강의실 선택 영역의 2행 구조로 바꿨습니다.
- 메시지와 알림 버튼에 44px 터치 영역과 접근 가능한 이름을 부여했습니다.
- 강의실 선택은 전역 헤더 한 곳으로 통합하고 피드의 중복 선택기를 제거했습니다.
- 모바일 다중 메일함 선택을 여러 줄 탭에서 한 줄 선택 상자로 바꿨습니다.
- PC 메일 목록·상세·작성 화면의 최대 폭을 512px에서 896px로 넓혔습니다.

## 왜 필요한가요?

긴 강의실 이름이 들어오면서 모바일 헤더 요소가 겹쳤고, 여러 메일함 탭은 작은 화면에서 줄바꿈되어 현재 선택지를 찾기 어려웠습니다. PC에서도 메일 본문이 지나치게 좁아 긴 내용을 읽고 작성하기 불편했습니다.

## 어떻게 검증했나요?

- Header, ClassroomSwitcher, MailboxPage, FeedPage 회귀 테스트 57개 통과
- Frontend 전체 테스트 256개 통과
- TypeScript 및 Vite Production build 성공
- 게시판 안에 별도 강의실 선택기가 렌더되지 않는 계약 확인

## 배운 점

반응형 UI는 모든 정보를 한 줄에 축소하기보다, 모바일에서 중요 컨텍스트를 별도 행으로 분리하는 편이 더 명확합니다. 전역 컨텍스트 선택기를 페이지마다 반복하면 현재 상태의 기준이 둘로 나뉘므로 단일 진입점을 유지해야 합니다.
