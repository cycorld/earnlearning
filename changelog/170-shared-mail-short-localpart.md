# 팀 메일 01@ ~ 15@ — 관리자 공용 메일함만 2글자 허용

## 무엇을 했나요?

부트캠프 팀 메일 `01@earnlearning.com` ~ `15@earnlearning.com`을 만들려는데 "최소 3자" 규칙에 걸렸습니다. 관리자가 만드는 **공용 메일함에 한해** 최소 2글자를 허용했습니다.

## 왜 이렇게 했나요?

- 학생/회사 신청은 그대로 최소 3자 — 짧은 주소는 희소 자원이라 선점 경쟁·혼동(1글자 오타 등)의 여지가 있어 일반 신청에는 풀지 않습니다.
- 관리자 공용 생성은 이미 금지어(hello, admin 등)도 예외인 "운영자 특권" 경로라, 같은 경로에 길이 완화를 얹는 게 규칙 체계상 자연스럽습니다.

## 어떻게 했나요?

검증 함수가 마침 두 갈래로 나뉘어 있었습니다 — 학생용 `ValidateLocalPart`(형식+금지어), 관리자용 `ValidateLocalPartFormat`(형식만). 관리자용에만 최소 2자 정규식을 적용하고, 학생용은 3자 정규식을 직접 쓰도록 분리했습니다. 회귀 테스트: 관리자 "01" 생성 OK / 관리자도 1자는 400 / 학생 "02" 신청은 여전히 400.

## 배운 점

검증 로직을 "누가 호출하느냐"별 함수로 나눠두면, 이런 정책 차등이 정규식 한 줄 차이로 끝납니다. 처음부터 한 함수에 boolean 플래그로 박았다면 호출부 전수 점검이 필요했을 것.
