# 진짜 이메일 주소가 생겼다 — Cloudflare 수신 + 개인·회사·공용 메일함

## 무엇을 했나요?

`원하는이름@earnlearning.com` 형태의 **진짜 이메일 주소**를 만들고, 언러닝 앱 안에 메일함을 넣었습니다.

- **세 종류 메일함**: 개인 주소(학생 신청), 회사 주소(회사 소유자가 등록), 공용 주소(관리자가 만들어 특정 학생들에게 공유 — hello@ 같은 금지어 주소도 관리자는 사용 가능).
- **관리자 승인제**: 주소를 신청하면 대기 상태 → 관리자가 승인해야 사용 가능. 반려되면 다른 이름으로 재신청. **승인 후에는 변경 불가** (이름은 바꿔도 이메일은 평생).
- 메일함 화면에서 접근 가능한 메일함을 선택(개인/회사/공용 배지)해서 각각의 받은편지함/보낸편지함, 첨부 다운로드, 답장·새 메일 발송.
- 새 메일이 오면 앱 알림 (공용 메일은 권한자 전원에게).
- 관리자 페이지에 승인 대기 큐 / 전체 계정 현황 / 공용 메일함·공유 관리 탭.
- 금지어 90여 개 기본 차단 (admin, billing, support, postmaster, …). 최소 3자.

## 왜 필요한가요?

창업 교육에서 학생들은 "회사"를 운영합니다. 진짜 회사처럼 외부와 이메일로 소통할 수 있으면 — 고객 인터뷰 요청, 설문 회신, 데모 신청 접수 — 시뮬레이션의 현실감이 확 올라갑니다. 지금까지 언러닝은 메일을 **보내기만** 했고(비밀번호 재설정 등, AWS SES), 받을 수는 없었습니다.

## 어떻게 만들었나요?

```
수신: 외부 메일 → MX(Cloudflare Email Routing, catch-all)
      → Email Worker (postal-mime 파싱, 첨부 base64)
      → POST /api/mail/inbound (공유 시크릿 인증)
      → SQLite 저장 + 알림 → 앱 메일함 UI
발송: 메일함 UI → 기존 AWS SES 경로 (In-Reply-To 헤더로 스레딩)
```

재미있는 설계 포인트들:

- **수신과 발송이 서로 다른 회사를 씁니다.** 발송은 기존 AWS SES 그대로, 수신만 Cloudflare Email Routing. MX 레코드는 수신에만 관여하고 SES는 자체 반송 도메인으로 SPF를 통과하기 때문에 충돌이 없습니다. 도메인에 MX가 원래 없었어서 깨끗하게 얹을 수 있었습니다.
- **Worker는 우체부, 백엔드가 집주인.** Cloudflare Worker는 메일을 파싱해 전달만 하고, "이 주소가 존재하는가"의 판단과 저장은 전부 LMS 백엔드가 합니다. 없는 주소면 백엔드가 404를 주고 Worker가 SMTP 반송(setReject)합니다. 데이터가 한 곳(SQLite)에 모여야 앱 알림·권한·백업이 기존 구조를 그대로 탑니다.
- **웹훅은 공유 시크릿으로 잠급니다.** 누구나 POST할 수 있는 공개 엔드포인트이므로 Worker와 백엔드가 같은 비밀값(`X-Mail-Webhook-Secret`)을 나눠 갖고 검증합니다.
- **HTML 메일은 텍스트로만 렌더링.** 외부에서 온 HTML을 그대로 그리면 XSS 위험이 있어 본문은 텍스트를 사용합니다.
- **답장 스레딩**: 원본 메일의 Message-ID를 In-Reply-To/References 헤더로 넣어 Gmail 등에서 대화가 한 줄로 묶입니다.

TDD로 진행: 주소 클레임 검증(형식·예약어·중복·불변), 웹훅 인증/미지정 주소 404/첨부 저장, 발송 스레딩, 소유자 격리(남의 메일 403) 테스트를 먼저 작성하고 구현했습니다.

**보안 점검도 별도로 했습니다.** 정적 코드 리뷰 + 실서버 침투 테스트로 전 엔드포인트의 접근 제어를 검증했고, 세 가지를 잡아 고쳤습니다:
1. OAuth 앱 토큰이 메일 API를 통과하던 구멍 — 메일은 내부 전용이라 OAuth 접근을 전면 차단 (`RejectOAuth` 미들웨어).
2. 발신자 표시명(유저/회사 이름은 변경 가능)을 RFC 5322로 인용해 `Ewha Admin <ceo@...>` 같은 발신자 위장 차단.
3. 발송기가 꺼져 있을 때 "보냈습니다"라고 거짓 성공하던 문제 — 이제 실패로 표시하고 보낸편지함에도 남기지 않습니다.

## 사용한 프롬프트

> earnlearning.com 으로 메일 발송하는건 aws ses 로 세팅되어 있지? 메일 수신 세팅 가능해? cloudflare 에 메일 수신 워커 세팅 후 메일함을 언러닝에 만들고 싶어.

## 배운 점

- 이메일 인프라는 "발송"과 "수신"이 완전히 별개 시스템입니다. 발송 평판(SPF/DKIM)과 수신 라우팅(MX)은 서로 다른 DNS 레코드로 움직여서, 발송은 AWS·수신은 Cloudflare 같은 조합이 자연스럽게 가능합니다.
- 외부 입력 중 이메일은 가장 지저분한 축입니다 — 봉투 주소와 헤더 주소가 다를 수 있고(스푸핑), 첨부 크기 폭탄이 올 수 있어 봉투 기준 + 크기 상한 + 반송 코드를 처음부터 설계해야 합니다.
