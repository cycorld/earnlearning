# 029. 정부과제 지원서 수정/삭제 및 승인 취소 기능

> **날짜**: 2026-04-01
> **태그**: `정부과제`, `UX`, `관리자`, `보상`

## 무엇을 했나요?

1. 학생이 승인 전 지원서를 **수정/삭제**할 수 있게 했습니다.
2. 관리자(교수)가 승인된 지원을 **취소**하고 보상금을 **회수**할 수 있게 했습니다.
3. 지원서 내용을 다른 학생들도 볼 수 있어 공부 자료로 활용 가능합니다.

## 왜 필요했나요?

- 학생이 지원서를 잘못 작성했을 때 수정할 방법이 없었습니다.
- 관리자가 실수로 빈 지원서를 승인한 경우 되돌릴 수 없었습니다.
- 다른 학생들의 지원서를 참고할 수 있으면 학습 효과가 높아집니다.

## 어떻게 만들었나요?

### 지원서 수정/삭제 API

```
PUT    /grants/:id/applications/:appId    # 본인 + pending만
DELETE /grants/:id/applications/:appId    # 본인 + pending만
```

- 본인 확인 + pending 상태 확인 후에만 수정/삭제 허용
- 승인(approved) 후에는 수정/삭제 차단

### 승인 취소 API

```
POST /admin/grants/:id/revoke/:appId    # 관리자 전용
```

- approved → pending으로 상태 변경
- 학생 지갑에서 보상금 debit, 관리자 지갑에 credit
- 학생에게 "승인 취소" 알림 발송

### 프론트엔드

- 본인 pending 지원서에 연필(수정)/휴지통(삭제) 아이콘 버튼
- 수정 시 인라인 마크다운 에디터 표시
- 관리자: 승인된 지원서에 빨간색 "승인 취소 (보상 회수)" 버튼
- 삭제/승인취소 시 confirm 다이얼로그로 실수 방지

## 배운 점

- **상태 기반 권한 제어**: `pending`/`approved` 상태에 따라 허용되는 작업이 달라집니다. 백엔드에서 반드시 검증해야 프론트엔드를 우회한 API 호출도 차단됩니다.
- **금융 트랜잭션 되돌리기**: 승인 시 debit/credit 순서와 취소 시 역순 debit/credit이 정확히 대칭이어야 잔액 불일치가 발생하지 않습니다.
