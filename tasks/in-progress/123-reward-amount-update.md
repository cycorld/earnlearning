## 설명
참여 보상 금액을 상향하고, 게시글 작성에 대한 신규 보상을 추가한다.

## 규칙 (변경/신규)
| 행동 | 보상 | 수령자 | 비고 |
|------|------|--------|------|
| 게시글 작성 | **+10,000원** (신규) | 작성자 본인 | 과제/자동 게시글 제외 |
| 게시글 삭제 | **−10,000원 회수** (신규) | 작성자 본인 | **관리자 삭제 시에도 회수** (어뷰징 방지) |
| 좋아요 | +500원 (10→500) | 글쓴이 | 자기 글 제외 |
| 좋아요 취소 | −500원 회수 | 글쓴이 | |
| 댓글 | +1,000원 (100→1,000) | 글쓴이 | 자기 글 제외 |
| 댓글 삭제 | −1,000원 회수 | 글쓴이 | |

## 작업 내용
- `wallet.TxPostReward = "post_reward"` 트랜잭션 타입 추가
- `PostRewardAmount = 10000` 상수 추가, `LikeRewardAmount` 10→500, `CommentRewardAmount` 100→1000
- `CreatePost`: 작성 성공 시 작성자에게 10,000원 credit (`uc.CreatePost` 경로만 → 과제/자동 게시글 제외)
- `DeletePost`: 삭제 성공 시 작성자에게서 10,000원 debit 회수 (작성자/관리자 삭제 모두)
- 기존 `engagement_reward_test.go` 단언 갱신(10→500, 100→1000)
- 신규 테스트: 게시글 작성 보상 / 작성자 삭제 회수 / 관리자 삭제 회수 / 과제·자동 게시글 보상 제외

## 알려진 한계 (후속 검토)
- 작성자가 보상금을 먼저 소진한 뒤 글을 삭제하면 회수 debit이 잔액부족으로 실패할 수 있음(기존 좋아요/댓글 회수와 동일한 특성). 강제 회수(음수 허용)는 경제 전반 영향이 있어 별도 티켓에서 검토.
