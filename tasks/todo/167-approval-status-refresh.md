## 제보 (2026-07-20 수업)
- admin이 가입 승인을 했는데도 학생 화면에는 계속 "승인 대기 중" 표시.
- 로그아웃 → 재로그인하면 해결되는 경우 다수. 재로그인 없이 자동 반영되게 해야 함.

## 원인 가설 (점검 필요)
- JWT 클레임에 status가 박혀 있어(`status":"approved` 형태) 발급 시점 상태가 클라이언트에 고정됨.
- 승인 대기 화면이 /auth/me 재조회 없이 로그인 시 캐시된 유저 상태만 봄.
- 승인 시점에 클라이언트로 상태 변경을 알릴 채널(WebSocket/폴링) 부재.

## 수정 방향(안)
1. 승인 대기 화면에서 주기 폴링(예: 10초) 또는 visibilitychange 시 `/auth/me` 재조회 → status 바뀌면 토큰 재발급(refresh) 후 자동 진입.
2. 백엔드: 승인 처리 시 해당 유저에게 WebSocket 이벤트/알림 발송 → 프론트가 받으면 즉시 갱신.
3. 미들웨어가 JWT status 클레임을 신뢰한다면 → 승인 직후 기존 토큰으로도 접근 가능하도록 DB 기준 판정으로 보정(또는 refresh 강제).

## 완료 기준
- 승인 대기 화면에 있는 학생이 재로그인 없이 승인 반영 (자동 진입).
- 회귀 테스트: 승인 전 pending 화면 → 승인 후 자동 전환.
